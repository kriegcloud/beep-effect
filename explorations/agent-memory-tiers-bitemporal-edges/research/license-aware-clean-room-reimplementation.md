### Legal mechanics governing each posture (primary sources)

- **No-license / unknown-license repo (harvest-mcp) = ALL RIGHTS RESERVED.** GitHub's own docs are explicit: "without a license, the default copyright laws apply, meaning that you retain all rights to your source code and no one may reproduce, distribute, or create derivative works from your work" (https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/licensing-a-repository). So an unlicensed repo is the **most** legally dangerous to copy from, not the least — there is no grant at all. Strategy must be: never copy a line; reimplement the *idea* of the taxonomy (which is likely unprotectable, see Altai below) under repo-native names.
- **Apache-2.0 (agentmemory) attribution mechanics — §4.** A distributed derivative must: (a) retain all copyright/patent/trademark/attribution notices from the source, (b) carry "prominent notices stating that You changed the files," and (c) **reproduce a `NOTICE` file's contents only if the original *has* a NOTICE file** (https://www.apache.org/licenses/LICENSE-2.0). **agentmemory has no root `NOTICE` file** (root listing shows LICENSE, README, AGENTS.md, CHANGELOG, etc. — no NOTICE) (https://github.com/rohitg00/agentmemory), so the conditional NOTICE-reproduction duty does not attach. Remaining concrete duties when porting agentmemory shapes: preserve "Copyright 2026 Rohit Ghumare" + the Apache-2.0 license text in a repo `THIRD_PARTY_NOTICES`/attribution file, and mark ported files as modified. Bonus over MIT: Apache **§3 grants an explicit patent license** from contributors, de-risking the retention/decay algorithms.
- **AGPL-3.0 (mike) — why "clean-room," and the AI trap.** A literal cross-language translation of a copyleft work is a **derivative work**, and "modify" includes adapting "all or part of the work in a fashion requiring copyright permission" (https://writing.kemitchell.com/2021/01/24/Reading-AGPL). If a port is a derivative of AGPL code, the **whole memory slice inherits AGPL** — and AGPL §13 adds the network/SaaS disclosure trigger (running it as a service forces source release). The legally-safe escape is clean-room: study only behavior/public docs/spec, firewall the implementer from the source (https://en.wikipedia.org/wiki/Clean-room_design). **Critical caution for this repo's workflow:** feeding mike's `schema.sql` to an LLM and asking it to "rewrite in Effect-Schema" likely does **not** satisfy the clean-room firewall — "using an AI trained on / shown the original code as the intermediary may not satisfy that standard, as no court has tested this fact pattern" (https://dev.to/pickuma/ai-license-laundering-how-code-generators-strip-open-source-obligations-2i0m). So mike's shapes must be re-derived from a behavioral *spec* (states pending→accepted/rejected; a version-provenance enum), never transcribed from its SQL.
- **What "borrowing a shape" actually risks — Google v. Oracle (2021).** SCOTUS ruled 6–2 that Google's copying of Java SE *declaring code* (names + structure/sequence/organization) was **fair use**, but it **expressly declined to decide whether that API code is copyrightable at all** (https://en.wikipedia.org/wiki/Google_LLC_v._Oracle_America,_Inc. ; opinion https://www.supremecourt.gov/opinions/20pdf/18-956_d18f.pdf). Takeaway: copying field-name/enum "shapes" verbatim is **not** a bright-line-safe act — it is a fact-specific fair-use gamble the Court left unresolved. Reimplementing under different (Effect-Schema-idiomatic) names sidesteps the question entirely.
- **Why the *abstract* shapes are nonetheless likely safe to reimplement — Computer Associates v. Altai (2d Cir. 1992).** The Abstraction-Filtration-Comparison test filters out, as unprotectable, elements dictated by efficiency or by external constraints — "merger" (when there is essentially one way to express an idea, idea and expression merge and copying is no bar) and "scènes à faire" (elements standard/indispensable to the work, dictated by hardware, **interoperability/compatibility requirements**, design standards, market demands, and standard programming techniques) (https://en.wikipedia.org/wiki/Computer_Associates_International,_Inc._v._Altai,_Inc. ; https://www.bitlaw.com/source/cases/copyright/altai.html). A four-tier memory enum, a salience×exp-decay retention score, a `tvalid/tvalidEnd` bitemporal edge, and a confidence heuristic are **functional ideas / standard techniques** — filtered out under Altai. The defensible line: reuse the *idea/algorithm*, reimplement the *expression* (exact name lists, code structure) natively.

### Reimplementation strategy by source (synthesis grounded in the above)

- **agentmemory (Apache-2.0) → PORT-WITH-ATTRIBUTION.** Lowest-friction source. May translate algorithm + shape directly, provided: (1) attribution file preserving "© 2026 Rohit Ghumare" + Apache-2.0 text, (2) ported files marked modified, (3) re-expressed in Effect-Schema. The iii-sdk/zod/Anthropic wiring is discarded; only `ConsolidationTier`, `SemanticMemory`, `GraphEdge`, `computeRetention`, `computeConfidence` shapes/algos cross over. Do **not** copy `parseTemporalGraphXml` (brittle regex, no char-span grounding — CAPTURE caution); reuse only the *prompt shape* and add EvidenceSpan char-span grounding.
- **mike (AGPL-3.0) → CLEAN-ROOM (spec-first, no source contact).** Write a behavioral spec of the two borrowed shapes — (a) candidate-edit gate states `pending|accepted|rejected` with source-span anchors, (b) version-lineage provenance enum distinguishing machine-proposed vs human-confirmed — from the *description*, then implement against the spec without the SQL in view, and without routing the SQL through an LLM. Re-name to repo idioms (do not transcribe `document_versions`/`document_edits` column names or the exact 6-value enum string list).
- **harvest-mcp (no/unknown license) → REIMPLEMENT FROM IDEA ONLY.** Treat as all-rights-reserved. The `ClassifiedParameter` provenance-`source` enum idea (heuristic|llm|manual|consistency_analysis) is the genuinely useful, transferable concept for the per-fact `source` enum on consolidated memory; reimplement it natively, do not copy types.
- **screenpipe (LicenseRef-Commercial) → REFERENCE PATTERN ONLY, REIMPLEMENT.** Most restrictive; do not vendor, embed, or copy. The namespaced-tag cross-store linking idea (`person:`/`project:`/`topic:` + `include_related` one-call expansion) is observed at the design level and rebuilt from scratch in Effect-Schema.

### Borrowed-shape → Effect-Schema mapping (design synthesis; targets are existing repo capabilities)

All borrowed shapes are **domain-neutral memory primitives** — none carry IP-law vocabulary — so porting them does not breach the epistemic slice's no-vocabulary / federation invariant (CAPTURE "keep zero IP-law vocabulary"; tree-snapshot confirms the gate enforces this). Concrete targets (from the live-tree snapshot + repo conventions):

- `ConsolidationTier = working|episodic|semantic|procedural` (agentmemory) → `LiteralKit` literal-union schema (repo's canonical literal-domain pattern), not a TS `type`/`enum`.
- `SemanticMemory { fact; confidence; sourceSessionIds[]; sourceMemoryIds[]; accessCount; strength }` (agentmemory) → `Model.Class` reusing existing `Confidence = UnitInterval` and EvidenceSpan char-span grounding (`packages/epistemic/domain/src/values/EvidenceSpan/...`) — confidence is **alreadyCovered**, do not rebuild.
- `GraphEdge { tvalid; tvalidEnd; version; supersededBy; isLatest; sourceObservationIds[]; context?; stale? }` (agentmemory) → net-new bitemporal-edge `Model.Class`, provenance fields aligned to `@beep/provenance` TextAnchor + `@beep/semantic-web` PROV-O substrate (tree-snapshot).
- `computeRetention = min(1, salience·exp(−λ·Δt) + reinforcementBoost)` + hot/warm/cold/evictable thresholds (agentmemory) → pure Effect function over the schema; algorithm is unprotectable (Altai).
- `computeConfidence` (base 0.5 + sharedSessions·0.1 cap .3 + recency; supersedes +0.1; contradicts −0.05; clamp [0,1]) (agentmemory) → pure Effect function feeding conflict/contradiction edges.
- mike candidate-gate states `pending|accepted|rejected` + version source enum → `LiteralKit` unions, **clean-room-derived**, landing as a **slice-local disposition axis** (`active`/`rejected`/`superseded`/`conflicted`) wired *beside* the forward-only `ClaimLifecycle` — **NOT** by extending `ClaimLifecycle` itself (corrected 2026-06-29 Codex gate-1: `ClaimLifecycle` is a shared kernel consumed by `CandidateClaim` + law-practice `Distinction`; widening it conflates workflow progress with truth state — see RESEARCH.md "Additive-only" invariant). Re-named, not transcribed.
- harvest-mcp `source: heuristic|llm|manual|consistency_analysis` → `LiteralKit` per-fact provenance-source union, reimplemented from concept.
- screenpipe namespaced tags `person:|project:|topic:` → repo-native uniform tag schema, reimplemented from pattern.
