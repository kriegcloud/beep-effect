# citation-grounding-governance-controls

Scope: the governance/ethical-wall controls wrapping the citation guard — output-side re-verification ladder, provenance-staleness handling, matter-isolation fencing, primary-source restriction, untrusted-document framing, and parse-once/charge-per-citation metering — and where each lands (this packet vs the agents/governance slices).

## Findings

### A. Output-side citation re-verification ladder (exact -> re-anchor nearest -> reject)

- The doc-haus pattern this packet is mining (`dochaus/plugin/legal.ts:214-232`, per CAPTURE.md) is a fail-closed ladder: `text.slice(charStart,charEnd)===excerpt ? verified : findQuote(text, excerpt, charStart) ? re-anchor {charStart,charEnd,excerpt,reanchored} : drop`. This is structurally identical to the W3C Web Annotation / Hypothesis "robust anchoring" fallback chain, which is the canonical, well-studied form of the same mechanism and verifies the design against an independent primary source.
- Hypothesis re-attaches a span by stepping through a four-strategy ladder, fastest/most-exact first, degrading to fuzzy: (1) RangeSelector — reapply stored XPath start/end; (2) PositionSelector — reapply stored global character offsets regardless of DOM; (3) context-first fuzzy match — fuzzy-locate stored prefix then suffix from the TextQuoteSelector; (4) selector-only fuzzy match — "simply run a fuzzy text search for the exact text … and see if we can come up with something similar enough." https://web.hypothes.is/blog/fuzzy-anchoring/
- The W3C Web Annotation Data Model encodes exactly the two-tier "exact offset vs recoverable quote" split the ladder needs: `TextPositionSelector` is "a pair of string offsets marking the start and end" (the exact tier), and `TextQuoteSelector` "has enough information that if the quoted text is still in the document, it can usually be found again through … a fuzzy full-text search" (the re-anchor tier). TextQuoteSelector is the W3C-intended recovery selector. https://web.hypothes.is/blog/fuzzy-anchoring/ and https://github.com/judell/TextQuoteAndPosition
- When all strategies fail Hypothesis "orphans" the annotation rather than mis-placing it — i.e. reject/drop rather than emit a near-match. The doc-haus rule "returns VERIFIED-with-location or NOT FOUND, never a near match" (CAPTURE.md doc-haus#2) is the same fail-closed terminal: the only outcomes are exact-verified, re-anchored-verified, or dropped — never a silent fuzzy substitution. https://web.hypothes.is/blog/fuzzy-anchoring/
- Repo reuse: the normalized->raw offset map and exact/lesser/fuzzy span ladder already exist MIT-clean in `@beep/langextract` Alignment (`findExact | findLesser via lowerWithSourceOffsets | findFuzzy Levenshtein`, per tree-snapshot + CAPTURE.md). The output-side guard should EXTEND Alignment as the re-anchor engine rather than port doc-haus (license unknown). LANDS IN THIS PACKET.

### B. Provenance staleness handling against the live source

- Two distinct staleness axes exist and must not be conflated. (1) **Byte-level live-file drift** — the cited span no longer occupies the recorded offsets because the source document changed; this is what the re-verification ladder re-checks "against the LIVE file" at output time (CAPTURE.md doc-haus#4). (2) **Authority-level staleness** — the cited case was correct when written but has since been reversed/overruled/distinguished. "A case may have existed and said exactly what was claimed at the time it was decided, but if it was subsequently reversed or distinguished, relying on it is as dangerous as citing a fake one." https://arxiv.org/html/2605.10186v1
- Authority staleness is the citator ("good law") problem: KeyCite/Shepard's/BCite flag negative treatment (reversed/overruled/superseded/distinguished) so a cite is still "good law." But citators are themselves unreliable — a William & Mary study found Shepard's and KeyCite missed or mislabeled ~1/3 of negative citations and BCite >2/3, and all three agreed on negative treatment in only 53 of 357 citing relationships (~85% disagreement). https://www.aallnet.org/wp-content/uploads/2018/12/LLJ_110n4_02_hellyer.pdf and https://infoguides.wtamu.edu/busi3312/shepardizing
- LegalCiteBench independently confirms the failure mode the guard exists to block: models are catastrophically bad at producing citations from memory (strongest model 6.80/100 on citation retrieval) but good at verifying a supplied one (error-detection 75.59, case-verification ~90). Crucially, their "Misleading Answer Rate" exceeds 94% on retrieval tasks for 20/21 models — i.e. models "confidently provide plausible-sounding but incorrect citations rather than expressing uncertainty." This is the quantitative justification for the ground-before-cite contract: never let the model author a cite, only verify one. https://arxiv.org/html/2605.10186v1
- Scoping: byte-level live-file re-anchoring LANDS IN THIS PACKET (it is the guard's output gate). Authority-level "good law" staleness is a citator/data-driver concern — it depends on CourtListener negative-treatment data and belongs with the gov-legal data drivers + `goals/official-data-sync-foundation`, surfaced to the guard as a provenance flag the guard can reject on. Keep the guard's staleness check to "does this verbatim text still exist at this provenance," and CONSUME an external "isGoodLaw" verdict rather than computing it here.

### C. Matter-isolation / confidentiality fencing (ethical wall)

- The canonical industry framing (Harvey, "Long Horizon Agents and Ethical Walls," 2026) makes the client matter the foundational structural unit and a security boundary enforced at three layers: "When Intapp says a wall exists between Matter A and Matter B, the AI platform enforces that wall at the retrieval layer, the context layer, and the output layer." https://www.harvey.ai/blog/long-horizon-agents-and-ethical-walls
- Fail-closed default for autonomous retrieval: "If an agent can't confirm that a document falls within the matter's boundary, it must skip that document and flag the uncertainty." Every interaction is scoped to one matter; agents "can only access resources within that matter's boundary." https://www.harvey.ai/blog/long-horizon-agents-and-ethical-walls
- Agent-memory contamination is the novel risk vs human screens: "If an agent working on Matter A picks up a fact, and then the same agent is later assigned to Matter B — which is on the other side of an ethical wall — does that prior context contaminate the new work?" This means provenance/citation memory MUST be matter-tagged so a verified span from Matter A can never be re-emitted into Matter B. https://www.harvey.ai/blog/long-horizon-agents-and-ethical-walls
- This duty is a hard professional obligation, not a nicety: ABA Formal Opinion 512 (Jul 29, 2024) holds that under Model Rule 1.6 a lawyer using generative AI must keep confidential all information relating to a representation "regardless of its source," must assess whether inputs may be "disclosed to or accessed by" others (incl. via self-learning models), and that boilerplate engagement-letter consent is inadequate. https://www.americanbar.org/news/abanews/aba-news-archives/2024/07/aba-issues-first-ethics-guidance-ai-tools/ and https://library.law.unc.edu/2025/02/aba-formal-opinion-512-the-paradigm-for-generative-ai-in-legal-practice/
- RAG access-control literature corroborates the retrieval-layer enforcement: if retrieval surfaces content the user is not authorized to view, the model "can faithfully summarize or quote it, producing a direct access control violation" — so fencing must be at retrieval (ABAC/ReBAC/fine-grained authz), not post-hoc on output. https://www.blockchain-council.org/ai/secure-rag-for-regulated-industries-data-privacy-access-control-prompt-injection-defense/
- Scoping: matter-isolation/ethical-wall fencing is cross-cutting AGENT GOVERNANCE, not citation-specific. Per tree-snapshot it maps to `goals/agent-governance-control-plane` (pending). DOES NOT LAND IN THIS PACKET as an owned capability; the guard COMPOSES it by requiring every EvidenceSpan/citation to carry a matter id and refusing to emit cross-matter provenance. The packet's only obligation is the matter-tag-on-span contract + fail-closed-if-untagged check.

### D. Official-primary-source webfetch restriction

- Legal-research doctrine: primary authority (constitutions, statutes, case law, administrative regs) is the only binding/mandatory authority; secondary sources (treatises, law-review, encyclopedias, and by extension AI summaries) are "never binding, but can be persuasive." A verification step must resolve to official primary records, never to a secondary or AI-generated restatement. https://uri.libguides.com/c.php?g=1311888&p=9642277 and https://www.law.georgetown.edu/wp-content/uploads/2018/02/secondarysources.pdf
- Operationalized, this is a webfetch/tool allowlist: citation re-verification against "live source" must target the authoritative record (e.g. CourtListener opinion text, GovInfo, eCFR, USPTO ODP) and treat anything else as evidence-of-existence only, not as groundable provenance. The Juris.AI#2 anti-pattern in CAPTURE.md (fabricated mock "sources" with no spans) is the explicit negative example of letting a non-primary, unspanned source masquerade as authority.
- Scoping: the *allowlist of reachable official sources* is a tool-governance / webfetch-policy concern that belongs with the agents slice (tool allowlists) and the gov-legal data drivers, NOT this packet. What LANDS HERE is the guard rule that a citation's groundable provenance must be an official primary record (a typed precondition on the provenance object), enforced regardless of which tool fetched it.

### E. Untrusted-document framing (indirect prompt injection)

- OWASP LLM01 (Prompt Injection, #1 on the 2025/2026 LLM Top 10) is the governing standard. Indirect injection "occurs when an LLM accepts input from external sources, such as websites or files," whose content "alters its behavior in unintended ways"; the root cause is the model's "inability to distinguish between trusted instructions and untrusted data." https://genai.owasp.org/llmrisk/llm01-prompt-injection/ and https://repello.ai/blog/owasp-llm-top-10-2026
- Canonical mitigation is segregation + fail-closed, not filtering: "Separate and clearly denote untrusted content to limit its influence on user prompts," apply semantic/string filters, and validate output groundedness. Source documents are data, never instructions. https://genai.owasp.org/llmrisk/llm01-prompt-injection/
- This directly reinforces the ground-before-cite contract (CAPTURE.md mike#1/research-squad#1): citations must derive only from opinion/passage text "supplied in this turn," and the output is rejected unless the non-citation text is byte-identical to the original — i.e. a document can never instruct the model to fabricate a cite, and any injected "instruction text" cannot survive the verbatim-equality check. Poison-RAG research shows a handful of malicious documents can drive >90% attacker-chosen answers, which the verbatim+span gate structurally defeats. https://genai.owasp.org/llmrisk/llm01-prompt-injection/
- Scoping: the *posture* (documents are untrusted data) is a foundational principle the guard CONSUMES, and the verbatim-equality gate is one concrete enforcement of it. But classifier/segregation defenses at ingest are an agents-slice context-assembly concern. The packet OWNS the verbatim-equality + span-existence gate; it does NOT own input sanitization/classification.

### F. Parse-once / charge-per-citation rate-and-cost metering

- CourtListener Citation Lookup API (the lookup driver, AGPL upstream) primary limits (v4): throttled to **60 valid citations per minute**; **at most 250 citations per single request** (citations past 250 "will be parsed, but not matched … the status key of such citations will be 429"); **64,000-character** text cap per request ("about 50 pages," requests over this "will be blocked for security"). https://wiki.free.law/c/courtlistener/help/api/rest/v4/citation-lookup and https://github.com/freelawproject/courtlistener/discussions/6895
- Two distinct 429 semantics: (1) per-citation `status: 429` inside the result array when a request exceeds the 250-citation cap (over-cap cites are parsed but unmatched); (2) HTTP 429 when the 60-valid/min throttle is exceeded, with a JSON body carrying an ISO-8601 datetime telling you when the next request is allowed. https://wiki.free.law/c/courtlistener/help/api/rest/v4/citation-lookup
- Result schema (each citation): `citation` (queried string), `normalized_citations` (corrected/canonical array), `start_index`/`end_index` (char positions in submitted text — the span primitive), `status` (HTTP-style outcome), `error_message`, `clusters` (matching case objects). https://wiki.free.law/c/courtlistener/help/api/rest/v4/citation-lookup
- "Parse-once / charge-per-citation" maps onto two standard patterns: a **token-bucket** limiter (tokens = the 60 valid-citations/min budget; deny + honor `wait_until` when empty; used by Stripe/Amazon for cost control because "every API call costs money") and **idempotency-key dedupe** (retry-safe "charge once per request"; e.g. Adyen requires a unique `idempotency-key` UUID so an action is performed only once). The guard should dedupe identical citation strings before metering so a citation parsed once is not re-charged across re-verification passes. https://www.hellointerview.com/learn/system-design/problem-breakdowns/distributed-rate-limiter and https://docs.adyen.com/development-resources/api-idempotency
- Scoping: parse-once/charge-per-citation metering of the lookup driver LANDS IN THIS PACKET (it is intrinsic to the citation-resolution lifecycle: batch <=250, chunk under 64k chars, token-bucket the 60/min, honor 429+wait, dedupe to parse-once). The generic distributed-rate-limiter infrastructure can be reused from any existing limiter rather than rebuilt.

### Scoping summary (this packet vs agents/governance)

- LANDS IN THIS PACKET: (A) output-side exact->re-anchor->reject ladder (extend langextract Alignment); (B) byte-level live-source staleness re-check + drop/re-anchor; (E-partial) the verbatim-equality + span-existence gate; (F) parse-once/charge-per-citation metering of the lookup driver; plus the *typed preconditions* the guard enforces — matter-tag-on-span, official-primary-source provenance type, isGoodLaw flag consumed.
- LANDS IN agents / `goals/agent-governance-control-plane` (pending): matter-isolation/ethical-wall retrieval+context+output fencing, fail-closed access, agent-memory matter-tagging, Intapp-Walls-style policy parity, confidentiality (ABA 512) policy.
