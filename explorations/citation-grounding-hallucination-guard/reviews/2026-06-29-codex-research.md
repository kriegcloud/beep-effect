# Codex research-gate critique — citation-grounding-hallucination-guard (2026-06-29)

## Blocking

- **Official-data routing is asserted as a ready home, but the cited goal does not currently cover court/reporter/citator data.** RESEARCH.md says `goals/official-data-sync-foundation` is the static-dataset ingest home for BSD `reporters-db`/`courts-db` and citator "good law" data (RESEARCH.md:210-212) and repeats the split as if settled (RESEARCH.md:319-321). The live goal exists and is active, but its manifest `sourceTargets` are only `iso4217`, `iana-media-types`, `iana-timezones`, and `cldr-territories` (goals/official-data-sync-foundation/ops/manifest.json:61-66), and the SPEC source policy names only those official feeds (goals/official-data-sync-foundation/SPEC.md:38-46). The optional tree snapshot routes court/jurisdiction/reporter vocab there as an attach-only recommendation (tree-snapshot.md:54), but that has not been reflected in the goal contract. **Fix:** either amend `official-data-sync-foundation` with explicit `reporters-db`, `courts-db`, and negative-treatment/citator source targets, or stop treating this as owned and create/route a separate `court-vocabulary-resolver` goal with an explicit dependency from this packet.

- **Matter-isolation is routed to a governance goal that does not implement product access-control fences.** RESEARCH.md claims `goals/agent-governance-control-plane` owns matter-isolation/ethical-wall fencing at retrieval, context, and output layers (RESEARCH.md:213-215; RESEARCH.md:287-289; RESEARCH.md:319-320). The live goal is pending, and all phases are pending (goals/agent-governance-control-plane/ops/manifest.json:37-85). More importantly, its SPEC scope is repo-wide agent governance, phase gates, prompt assets, and law-to-command validation (goals/agent-governance-control-plane/SPEC.md:101-115), not product authorization, matter-scoped retrieval, context assembly, or output filtering. This is a feasibility hole for a confidentiality-critical guard. **Fix:** create or attach to a concrete product/security goal with target packages, matter model, retrieval filters, context builder enforcement, output validator, audit trail, and fail-closed tests; leave the generic governance packet as policy inheritance only.

- **The matter-id precondition has no current model carrier.** RESEARCH.md says every `EvidenceSpan`/citation must carry a matter id and untagged spans must be refused (RESEARCH.md:287-289). The actual `EvidenceSpan` model is only `TextAnchorFields` plus `confidence` (packages/epistemic/domain/src/values/EvidenceSpan/EvidenceSpan.model.ts:76-80), and `TextAnchorFields` are only `startChar`, `endChar`, and `quote` (packages/foundation/modeling/provenance/src/TextAnchor.ts:37-40). The cited `@beep/nlp` provenance record has `source`, `generatedBy`, `timestamp`, and optional confidence, but no matter or wall boundary (packages/foundation/capability/nlp/src/Handoff/Contract.ts:225-236). **Fix:** define a `CitationEvidence`/`MatterScopedEvidenceSpan` wrapper or provenance extension that carries `matterId`, `sourceDocumentId`, authority/source class, and wall policy metadata before claiming the precondition is enforceable.

- **`EvidenceSpan` is not a 1:1 internal model for Anthropic citations.** RESEARCH.md says `EvidenceSpan` "matches Anthropic's `{startChar,endChar,cited_text}` shape 1:1" and should be adopted as the internal grounded-answer model (RESEARCH.md:167-169). It only represents a char span plus quote/confidence (packages/epistemic/domain/src/values/EvidenceSpan/EvidenceSpan.model.ts:52-80). It lacks `document_index`/source identity, title, `char_location | page_location | content_block_location`, page/content-block variants, and the matter/source-trust fields the same RESEARCH later requires (RESEARCH.md:287-292). **Fix:** treat `EvidenceSpan` as the char-anchor primitive only; add a citation-location tagged union and document/provenance envelope around it.

- **The TextAnchor "invariant" is documentation, not a schema-enforced guarantee.** RESEARCH.md calls `@beep/provenance` `TextAnchor` an invariant where `text.slice(startChar,endChar)` reproduces `quote` (RESEARCH.md:164-166). The source describes the intended invariant in prose (packages/foundation/modeling/provenance/src/TextAnchor.ts:4-9), but the schema only exposes the three fields (packages/foundation/modeling/provenance/src/TextAnchor.ts:37-40); the only validator is `isWellOrdered`, which checks `startChar <= endChar` and explicitly says schema-level cross-field enforcement is a follow-up (packages/foundation/modeling/provenance/src/TextAnchor.ts:70-91). For a hallucination guard, this cannot be treated as a proof. **Fix:** add a verified constructor/decoder that takes the source text or source digest and fails unless the slice exactly matches the quote; persist a verification timestamp/source digest if live-file drift matters.

## Advisory

- **The inventory misses `@beep/file-processing` as the source-text extraction substrate.** RESEARCH.md says the repo owns the grounding substrate but lists only langextract/provenance/epistemic/nlp span primitives (RESEARCH.md:150-171), then declares cross-page/page source handling net-new (RESEARCH.md:230-236). `@beep/file-processing` exists (packages/foundation/capability/file-processing/package.json:1-5), supports `pdf-text-layer`, `plain-text`, markdown, Office formats, and more (packages/foundation/capability/file-processing/src/Strategy/index.ts:82-97; packages/foundation/capability/file-processing/src/Strategy/index.ts:407-423), and has `TextSpan {startOffset,endOffset,text}` plus extraction results (packages/foundation/capability/file-processing/src/Extraction/index.ts:68-118). It does not solve legal citations, but it is a better existing ingestion boundary than a fresh source-text abstraction. **Fix:** add `@beep/file-processing` to the capability inventory and route local privileged document text extraction through it.

- **The inventory misses `@beep/nlp` `TextChunk`/`AnnotatedDocument` for chunk-global mapping.** RESEARCH.md cites only `@beep/nlp` `Contract.Span` as a canonical handoff span (RESEARCH.md:170-171) while the same document says cross-chunk straddle needs a `chunk -> [globalStart,globalEnd)` map (RESEARCH.md:104-108; RESEARCH.md:230-232). `@beep/nlp` already models `TextChunk` with `id`, `kind`, `provenance`, `span`, and `text`, and `AnnotatedDocument` carries a chunk array (packages/foundation/capability/nlp/src/Handoff/Contract.ts:252-263; packages/foundation/capability/nlp/src/Handoff/Contract.ts:367-378). **Fix:** extend or adapt `TextChunk`/`AnnotatedDocument` for global chunk maps before declaring a bespoke chunk reconstruction model.

- **The Lexical gap is only partial; a reusable schema/plain-text substrate exists.** RESEARCH.md says no plain-to-markup/Lexical citation annotation round-trip exists (RESEARCH.md:235-236). That is true for citation annotation, but the inventory omits `@beep/lexical-schema`, which models serialized editor state (packages/foundation/modeling/lexical/package.json:1-5), serialized text nodes (packages/foundation/modeling/lexical/src/Lexical.model.ts:989-1019), and `nodeToPlainText` / `editorStateToPlainText` helpers (packages/foundation/modeling/lexical/src/Lexical.model.ts:2222-2277). **Fix:** reclassify this as "partial substrate found": annotation/range mutation is net-new, but serialized-state parsing and plain-text projection are existing bricks.

- **The structured-output implementation path misses existing in-repo prior art.** RESEARCH.md correctly notes native Anthropic citations and strict JSON output are incompatible in one call, and recommends a separate structured pass (RESEARCH.md:266-271). It does not cite the repo's existing forced-tool structured-output and repair machinery: `@beep/agents-server` streams Anthropic forced-tool JSON blocks, validates each slice, holds invalid blocks, and repairs them (packages/agents/server/src/AssistantTurn/AnthropicTurnKernel.ts:52-70; packages/agents/server/src/AssistantTurn/AnthropicTurnKernel.ts:115-167), with provider-specific codecs (packages/agents/server/src/AssistantTurn/AnthropicTurnCodec.ts:1-14; packages/agents/server/src/AssistantTurn/AnthropicTurnCodec.ts:123-154). **Fix:** list `@beep/agents-server` as implementation prior art for the separate non-citation structured pass and validation/repair loop.

- **The `wait_until` field is stated as fact despite being an open wire-shape risk.** RESEARCH.md says the minute throttle response carries `wait_until` (RESEARCH.md:299-304), then later lists `wait_until` vs `wait_util` as an open verification item (RESEARCH.md:332-335). The raw governance dossier also says the v4 doc render returned `wait_util` (research/citation-grounding-governance-controls.md:78-82). This is not safe as a decoder contract. **Fix:** move the field spelling out of factual constraints, decode both spellings behind a compatibility codec, and verify against a real 429 or source serializer before implementation.

- **"Verbatim-equality gate structurally defeats poison-RAG injection" is too broad.** RESEARCH.md says documents are untrusted data and the verbatim-equality gate structurally defeats poison-RAG injection (RESEARCH.md:140-142). Verbatim verification proves a quoted span exists; it does not prevent an injected document from steering answer composition, tool choice, retrieval, or non-quoted prose. The raw governance dossier itself says classifier/segregation defenses live outside this packet (research/citation-grounding-governance-controls.md:37-42). **Fix:** narrow the claim to "prevents fabricated quote text and unspanned citations" and keep prompt-injection defenses as retrieval/context/tool-policy requirements.

## Confirmed sound

- **No fabricated `@beep/*` package paths were found in RESEARCH.md.** The cited package names resolve in live `package.json` files (`@beep/langextract`, `@beep/provenance`, `@beep/epistemic-domain`, `@beep/nlp`, `@beep/shared-domain`, `@beep/courtlistener`, `@beep/anthropic`, `@beep/law-practice-domain`, `@beep/schema`), and the exact cited file/directory paths all exist. This directly supports the path inventory at RESEARCH.md:156-205.

- **The core legal-citation parser/resolution gaps are real.** Targeted package-source search for `eyecite`, `reporters-db`, `courts-db`, `CitationResolution`, `UnmatchedCitation`, `postCitationLookup`, `normalized_citations`, and full/short/supra/id citation classes returned no package-source hits; a broader `Citation` search only surfaced unrelated generated ACP elicitation symbols and Box `AiCitation`. The NOT FOUND claims for legal citation parser, `CitationResolution`/`UnmatchedCitation`, and CourtListener lookup wrapper are therefore sound (RESEARCH.md:219-229).

- **`@beep/courtlistener` being a bare skeleton is verified.** RESEARCH.md says it is `VERSION = "0.0.0"` only (RESEARCH.md:195-198; RESEARCH.md:225-227). The package has one source file, and `packages/drivers/courtlistener/src/index.ts` exports only `VERSION` (packages/drivers/courtlistener/src/index.ts:1-11).

- **The `@beep/langextract` alignment description and "not enough for verbatim quotes" caution are sound.** RESEARCH.md describes exact, case-folded lesser, fuzzy, and unaligned statuses (RESEARCH.md:156-160) and warns not to promote fuzzy/case-folded matches to legal quote verification (RESEARCH.md:95-100; RESEARCH.md:328-330). The implementation matches: `findExact` uses `indexOf`, `findLesser` lowercases with offset arrays, `findFuzzy` runs a Levenshtein word-window, and `alignCandidate` emits `match_exact`, `match_lesser`, `match_fuzzy`, or `unaligned` (packages/foundation/capability/langextract/src/Alignment/index.ts:77-128; packages/foundation/capability/langextract/src/Alignment/index.ts:172-203; packages/foundation/capability/langextract/src/Alignment/index.ts:231-246).

- **The licensing correction is directionally sound and adequately flagged.** RESEARCH.md corrects CAPTURE's eyecite AGPL premise by distinguishing BSD `eyecite`/`reporters-db`/`courts-db`/`eyecite-js` from AGPL CourtListener app glue (RESEARCH.md:25-31; RESEARCH.md:242-254), flags `mike` as AGPL (RESEARCH.md:242-248), flags doc-haus as unknown-license (RESEARCH.md:257-259), and flags `research-squad` prompt text as unverified-license (RESEARCH.md:260-262). I did not find an additional unflagged GPL/AGPL dependency proposal in the reviewed RESEARCH.md.

- **The Anthropic citations vs strict JSON constraint is a real implementation gotcha.** RESEARCH.md says citations require interleaved text blocks and cannot be combined with strict structured output in one call (RESEARCH.md:266-271). The raw dossier makes the same point and recommends deriving JSON downstream or running a separate non-citation pass (research/ground-before-cite-contract-design.md:26-30). That is the right architectural warning; the missing part is only the in-repo `@beep/agents-server` prior art noted above.
