# shared-effect-http-client-layer

Scope: design a reusable Effect (v4 / `effect/unstable/http`) HttpClient layer factory — Redacted auth, Retry-After honoring, FIFO token-bucket throttle, TTL-jitter cache, typed status→error mapping — generalizing the us-gov-open-data-mcp + TalentScore nuggets, grounded in the version this repo actually ships.

## Findings

### Version anchor — this repo is Effect v4, NOT classic `@effect/platform`

- The repo pins `effect@4.0.0-beta.91` (root `package.json`; drivers use `catalog:`), and HTTP lives at `effect/unstable/http`, e.g. `import { FetchHttpClient } from "effect/unstable/http"; import * as HttpClient from "effect/unstable/http/HttpClient"` (source: `/home/elpresidank/YeeBois/projects/beep-effect/packages/drivers/uspto/src/Uspto.service.ts:11-21`, root `/home/elpresidank/YeeBois/projects/beep-effect/package.json`). **Consequence: the v3 docs at `effect-ts.github.io/effect` and most blog patterns (`@effect/platform`, `effect/RateLimiter`) do not match the installed API surface.** The authoritative reference is the installed `node_modules/effect/dist/**` (cited throughout below). The would-be v4 doc at `mintlify.com/Effect-TS/effect-smol/unstable/http-client` currently returns HTTP 410 Gone (verified 2026-06-29), so local `.d.ts`/source is the only reliable spec.
- The `effect/unstable/http` barrel re-exports `FetchHttpClient`, `Headers`, `HttpClient`, `HttpClientError`, `HttpClientRequest`, `HttpClientResponse`, `UrlParams` (source: `node_modules/effect/dist/unstable/http/index.d.ts:15-115`). HttpApi (the declarative contract layer govinfo already uses) is a *separate* module, `effect/unstable/httpapi` (`HttpApiEndpoint`, `HttpApiSchema`, `HttpApiClient`) (source: `packages/drivers/govinfo/src/domain/contracts/Search/Search.http.ts:2`, `node_modules/effect/dist/unstable/httpapi/`).

### Base client + Redacted auth (the TalentScore #7 pattern, in v4 form)

- Transport layer: `FetchHttpClient.layer: Layer.Layer<HttpClient.HttpClient>`, built on the Web Fetch API; the `fetch` impl is injected via a `Fetch` `Context.Reference` (test seam) (source: `node_modules/effect/src/unstable/http/FetchHttpClient.ts:30,123`).
- The canonical "wrap a base client" pipeline composes `HttpClient.mapRequest(...)` (prepend base URL + inject headers) → `HttpClient.filterStatusOk` (fail non-2xx) → retry/rate-limit. All three combinators exist in v4: `mapRequest`, `mapRequestEffect`, `filterStatusOk` (`<E,R>(self) => HttpClient.With<E | HttpClientError, R>`), `transformResponse`, `tapRequest`, `catchTag`, `catchTags`, `retry`, `retryTransient`, `withCookiesRef`, `followRedirects`, `make` (source: `node_modules/effect/dist/unstable/http/HttpClient.d.ts:200-887`).
- Request shaping helpers for the per-driver auth matrix: `HttpClientRequest.prependUrl` / `appendUrl` / `setUrl` (version-pinned base URL), `setHeader` / `setHeaders`, `bearerToken` (Bearer), `acceptJson`, `bodyJson`, `schemaBodyJson` (source: `node_modules/effect/src/unstable/http/HttpClientRequest.ts:294-977`). Query-param auth (api.data.gov `api_key` for govinfo/DOL) uses the `UrlParams` module.
- Secret handling: `Redacted.make(value, options?)` wraps a secret that hides itself from logs/`toString`; `Redacted.value(self)` extracts at the call site (source: `node_modules/effect/dist/Redacted.d.ts:175,200`). This is exactly the TalentScore nugget: `HttpClientRequest.setHeader(req, "x-api-key", Redacted.value(key))`. The repo already does this in USPTO: `HttpClientRequest.setHeader(request, "X-API-KEY", Redacted.value(key))` with `apiKey: O.Option<Redacted.Redacted<string>>` (source: `packages/drivers/uspto/src/Uspto.service.ts:199,253`).
- Env→Redacted loading: `Config.redacted(name?): Config<Redacted<string>>` — documented as "Shortcut for `Config.schema(Schema.Redacted(Schema.String), name)`", parses to a `Redacted` container hidden from logs (source: `node_modules/effect/dist/Config.d.ts:1095-1134`). Schema-side alternative `S.RedactedFromValue` is used in `packages/drivers/uspto/src/Uspto.config.ts:45`. **Design: the factory takes `Config.redacted("<ENV>")` per driver and conditionally injects header/query/body auth; absent key → omit auth (graceful), matching the us-gov-open-data-mcp `envParams` "graceful omit".**

### Typed status→error mapping (patent-search-mcp-server #6 / harvest-mcp #8, in v4 form)

- v4's HTTP error model is a single `HttpClientError` (`_tag: "HttpClientError"`) carrying a `reason: HttpClientErrorReason` discriminated union; the reason tags are `TransportError`, `EncodeError`, `InvalidUrlError`, `StatusCodeError`, `DecodeError`, `EmptyBodyError` (source: `node_modules/effect/dist/unstable/http/HttpClientError.d.ts:21-185`). **This differs from v3's `RequestError`/`ResponseError` naming — do not port v3 examples verbatim.**
- `StatusCodeError` is the status→error hook: it carries `{ request, response, cause?, description? }`, i.e. the full `HttpClientResponse` (so `.response.status`, `.response.headers` are available) plus `methodAndUrl`/`message` getters (source: `node_modules/effect/dist/unstable/http/HttpClientError.d.ts:128-156`). There is also a schema-encodable `HttpClientErrorSchema` (`_tag: "HttpError"`) for serializing across the MCP boundary (source: same file, lines 227-245).
- **Design: map status → a `Schema.TaggedError` (beep convention) inside `HttpClient.catchTag("HttpClientError", ...)` or `transformResponse`, switching on `reason._tag === "StatusCodeError"` and `reason.response.status`.** The patent-search-mcp-server status table (401→`unauthenticated`, 402→`payment_required`, 403→`permission_denied`, 429→`rate_limited`, ≥500→`server_error`, else→`bad_request`) ports directly as the `Match`/`codeForStatus` body (nugget `patent-search-mcp-server#6`, CAPTURE.md line 49). The repo precedent for status→domain-error is `M365Error.fromReason("throttled", { status, retryAfterSeconds })` (source: `packages/drivers/m365/src/M365.service.ts:787-793`, `M365.errors.ts:104-134`).

### Retry + Retry-After honoring

- `HttpClient.retryTransient(options)` is native and retries transport errors, timeouts, and transient *responses* with a configurable `Schedule`. Options: `{ retryOn?: "errors-only" | "response-only" | "errors-and-responses"; while?: Predicate; schedule?: Schedule; times?: number }` (source: `node_modules/effect/dist/unstable/http/HttpClient.d.ts:602-640`).
- Its built-in transient classifier treats these statuses as transient: **408, 429, 500, 502, 503, 504**; for errors it retries on `TransportError` and timeouts: `isTransientError = Cause.isTimeoutError(error) || (reason._tag === "TransportError" || (reason._tag === "StatusCodeError" && isTransientResponse(reason.response)))` (source: `node_modules/effect/src/unstable/http/HttpClient.ts:2065-2090`). The TalentScore nugget `retryTransient({ times: 3, schedule: Schedule.exponential("250 millis", 1.5) })` works as-is.
- Backoff schedules available in v4: `Schedule.exponential(base, factor?)`, `Schedule.jittered(self)` (adds randomization to any schedule — distinct from cache TTL jitter), `Schedule.spaced`, `Schedule.recurs`, `Schedule.addDelay`, `Schedule.passthrough` (source: `node_modules/effect/dist/Schedule.d.ts:480,3172,3623,3653,3723,4120`).
- **`retryTransient` does NOT read `Retry-After` to set the delay** — it follows the provided `Schedule` only (the schedule input can be the response, but no built-in reads the header). Two ways to honor `Retry-After`: (a) the native `HttpClient.withRateLimiter` (below) learns from `Retry-After` automatically; (b) the in-repo manual pattern: `executeWithRetry` reads `response.headers["retry-after"]` on 429/503, sleeps that many seconds, then recurses (source: `packages/drivers/m365/src/M365.service.ts:769-861`). RFC 9110 §10.2.3 defines `Retry-After = HTTP-date / delay-seconds` (two forms; `delay-seconds = 1*DIGIT`), applicable to 503/429 (and 3xx) (source: https://www.rfc-editor.org/rfc/rfc9110.html#name-retry-after). **Note: the m365 in-repo helper only parses the numeric `delay-seconds` form, not `HTTP-date` — a gap to close in the generalized layer.**

### Rate-limit / throttle — v4 ships this natively (big design simplification)

- **There is NO top-level in-memory `RateLimiter` module in v4 beta.91** (the v3 `effect/RateLimiter` token-bucket/fixed-window is gone). The only `RateLimiter` is `effect/unstable/persistence/RateLimiter`, a store-backed quota service (source: `node_modules/effect/dist/unstable/persistence/RateLimiter.d.ts`; confirmed absent elsewhere via `grep -rl RateLimiter node_modules/effect/dist`). v3 contrast: https://effect-ts.github.io/effect/effect/RateLimiter.ts.html.
- `RateLimiter` (persistence) supports `algorithm?: "fixed-window" | "token-bucket"`, `window: Duration.Input`, `limit: number`, optional per-call `tokens?: number` (variable cost). `make: Effect<RateLimiter, never, RateLimiterStore>` and `layer: Layer<RateLimiter, never, RateLimiterStore>`; it runs fully in-process via `RateLimiter.layerStoreMemory: Layer<RateLimiterStore>` ("process-local in-memory"), or `layerStoreRedis` for shared limits (source: `node_modules/effect/dist/unstable/persistence/RateLimiter.d.ts:41-149,418,480`).
- **`HttpClient.withRateLimiter` is a native combinator that replaces most of the us-gov-open-data-mcp hand-rolled token bucket.** It (1) applies the `RateLimiter` per request, (2) "can update limits by inspecting common rate limit response headers," and (3) "automatically retries HTTP 429 responses ... by forcing the retry back through the limiter" (source: `node_modules/effect/src/unstable/http/HttpClient.ts:1310-1360`). Options `WithRateLimiter.Options`: `{ limiter, window, limit, key: string | (req=>string), algorithm?, tokens?: number | (req=>number), disableResponseInspection?, disableAdaptiveLearning? }`; error channel gains `RateLimiter.RateLimiterError` (source: `node_modules/effect/src/unstable/http/HttpClient.ts:1271-1308`). The `key`/`tokens` functions cover the courtlistener "charge N tokens per citation" nugget natively.
- **`withRateLimiter` honors `Retry-After` (both RFC forms) and the IETF RateLimit headers.** Its `parseRetryAfter` parses the numeric `delay-seconds` form *and* the `HTTP-date` form (`Date.parse`, delta vs `clock`), clamping to ≥1ms (source: `node_modules/effect/src/unstable/http/HttpClient.ts:1584-1606`). It also reads `ratelimit-limit`/`x-ratelimit-limit`, `ratelimit-remaining`/`x-ratelimit-remaining`, `ratelimit-reset`/`x-ratelimit-reset`, `ratelimit-reset-after`/`x-ratelimit-reset-after` to adapt limits (source: same file, lines 1426,1555-1581) — the IETF `draft-ietf-httpapi-ratelimit-headers` family (currently draft-11, May 2026: https://datatracker.ietf.org/doc/draft-ietf-httpapi-ratelimit-headers/). `disableAdaptiveLearning` turns off the `Retry-After` learning specifically.
- **FIFO fairness primitive (if strict ordering is needed beyond the quota limiter):** Effect's `Semaphore` is documented FIFO — "Concurrent pending `take` calls are processed in a first-in, first-out manner" (source: `node_modules/effect/src/Semaphore.ts:123`, `node_modules/effect/dist/Semaphore.d.ts:111`). API: `Semaphore.make(permits): Effect<Semaphore>`, `withPermits(n)(effect)`, `withPermitsIfAvailable(n)(effect)` (fast-path / non-blocking — the us-gov-open-data-mcp "fast path when token free"), `makeUnsafe(permits)` (source: `node_modules/effect/dist/Semaphore.d.ts:175-437`). The us-gov-open-data-mcp single-timer FIFO-drain token bucket maps to `Semaphore(burst)` + a refill fiber, but **the persistence `RateLimiter` token-bucket + `withRateLimiter` is the lower-effort path** and already does Retry-After + 429 retry; reserve a hand-built Semaphore throttle only if you need strict request *ordering* (the persistence limiter sleeps each fiber by computed delay and does not guarantee FIFO dispatch — UNVERIFIED whether its waiters are ordered).

### TTL-jitter KV cache (uspto-patents-mcp #6, in v4 form)

- v4 ships `Cache` (top-level `effect/Cache`) and `ScopedCache` (for resource-bearing values). `Cache.make({ lookup, capacity: number, timeToLive?: Duration.Input })` is the simple form (static TTL). `Cache.makeWith({ lookup, capacity, timeToLive?: (exit: Exit<A,E>, key: Key) => Duration.Input })` is the **jitter + negative-caching hook**: `timeToLive` is a function of both the `Exit` (success vs failure) and the key (source: `node_modules/effect/dist/Cache.d.ts:84-227`). **Design: implement the uspto-patents-mcp "+10% random TTL jitter to avoid stampede" inside `makeWith`'s `timeToLive` (e.g. `base + Random*0.1*base`), and a shorter TTL for failure `Exit`s (negative caching) — neither is possible with the static `make` form.** The repo already uses this module: `@beep/nlp` wraps each backend op in an `effect/Cache` with `capacity` + `timeToLive` via `Cache.make` (source: `packages/foundation/capability/nlp/src/Backend/Composition.ts:21-25,7-8`).
- The stable cache-key (uspto-patents-mcp `stableKey()` = sorted-key serialization) has no Effect built-in; port the deterministic sorted-key join as a small helper. The us-gov-open-data-mcp variant cached to *disk*; for Effect, in-process `Cache`/`ScopedCache` is the default, and `effect/unstable/persistence/PersistedCache` exists if cross-process/durable caching is wanted (source: `node_modules/effect/dist/unstable/persistence/index.d.ts:15`).

### Generalization seam — how the shared layer plugs into every driver

- The factory's output is a transformed `HttpClient.HttpClient` (a `Layer<HttpClient.HttpClient>` or a `(client) => client` function). For hand-rolled drivers (CourtListener/eCFR/DOL/FedReg) it is consumed directly from context. For the **HttpApi-contract drivers (govinfo already has a `Search` HttpApi)**, `HttpApiClient.make(api, { transformClient?: (client: HttpClient) => HttpClient, baseUrl? })` derives a typed client and accepts the shared layer via `transformClient` (or just reads the provided `HttpClient.HttpClient` from context) (source: `node_modules/effect/dist/unstable/httpapi/HttpApiClient.d.ts:128-140`). **This `transformClient` hook is the single generalization point: one shared `auth+retry+cache+rateLimit` transformer, applied either as the context `HttpClient` layer or as `transformClient`, works for both the raw-client and HttpApi-client drivers.**
- Recommended composition order (outermost→innermost effect): `FetchHttpClient.layer` → `mapRequest`(prepend versioned base URL + Redacted auth header/query) → `filterStatusOk` → `catchTag`(StatusCodeError→`Schema.TaggedError`) → `retryTransient`({times, schedule: `Schedule.exponential` ⊕ `Schedule.jittered`}) for 5xx/timeouts → `withRateLimiter`({limiter, algorithm:"token-bucket", key, tokens}) for 429 + Retry-After + quota; wrap the per-request fetch fn in `Cache.makeWith` (TTL-jitter) at the driver method boundary (GET/idempotent only). Provide `RateLimiter.layer` + `RateLimiter.layerStoreMemory` once at the runtime.

## Open / Unverified

- **Persistence `RateLimiter` FIFO ordering:** the token-bucket limiter sleeps each fiber by a computed delay; whether concurrent waiters are dispatched in strict arrival order (the us-gov-open-data-mcp FIFO property) is UNVERIFIED. If strict FIFO matters, layer a `Semaphore` (documented FIFO) over the limiter.
- **`withRateLimiter` exact header-update semantics:** confirmed it parses `Retry-After` + `RateLimit-*`/`X-RateLimit-*`, but the precise rule for how it mutates `limit`/`window` from `remaining`/`reset` (vs. just delaying) was read from source comments, not exercised — validate behavior in an integration test before relying on adaptive learning in production.
- **Effect v4 RateLimiter relocation history:** corroborated only by the absence in installed `node_modules` vs. the v3 doc URL; no changelog entry was fetched. Treat "removed from top-level, moved to persistence" as installed-version-true, not necessarily a documented deprecation.
- **CourtListener "anonymous = 401":** CAPTURE.md asserts V4 *enforces* auth (anonymous→401), but the FLP wiki states "many of CourtListener's APIs are open by default" (lower unauthenticated rate limits). These conflict; the precise per-endpoint auth-enforcement matrix is UNVERIFIED and affects key-gated vs always-on MCP registration (sibling subtopic), not the client-layer mechanics.
- **mintlify effect-smol HTTP-client doc** (`mintlify.com/Effect-TS/effect-smol/unstable/http-client`) returned **HTTP 410 Gone** (2026-06-29) — no canonical hosted v4 doc was reachable; all v4 API claims rest on installed source.
- **TalentScore source** (`packages/server/src/public/files/upload-thing-api.ts`) is a private/external repo not inspected here; the pattern is taken from the CAPTURE nugget and reconciled against the v4 API, not the original file.
