# Prompt Assets

These prompt assets are reusable building blocks for future validator, implementer, and downstream-initiative sessions.

## Prompt Set

- [VALIDATION_ORCHESTRATOR_PROMPT.md](./VALIDATION_ORCHESTRATOR_PROMPT.md) — generic repo-governance validator prompt
- [IMPLEMENTATION_EXECUTOR_PROMPT.md](./IMPLEMENTATION_EXECUTOR_PROMPT.md) — generic packet-driven implementation prompt
- [AUDITOR_CATALOG_PROMPT.md](./AUDITOR_CATALOG_PROMPT.md) — adversarial sub-agent creation prompt
- [CATEGORY_WORKFLOW_TEMPLATE.md](./CATEGORY_WORKFLOW_TEMPLATE.md) — reusable category-loop template
- [CONSUMER_SPEC_BOOTSTRAP_TEMPLATE.md](./CONSUMER_SPEC_BOOTSTRAP_TEMPLATE.md) — template for new downstream initiatives

## Prompt Rule

These assets are required to stay generic. Downstream initiatives are required to add domain overlays instead of editing the generic governance contract directly.
