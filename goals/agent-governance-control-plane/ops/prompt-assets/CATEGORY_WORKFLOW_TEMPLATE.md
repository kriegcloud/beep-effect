# Category Workflow Template

Use this template for any control-plane category or consumer-spec category that inherits the governance package.

## Category Loop

1. `Research`
   - inspect repo reality
   - inspect prior outputs
   - inspect reusable precedents
2. `Plan`
   - lock acceptance criteria
   - lock contracts and interfaces
   - lock stop conditions
3. `Implement`
   - write or refine the category artifacts
4. `Refine`
   - run adversarial auditors
   - close blockers
5. `Validate`
   - emit explicit evidence of closure

## Required Artifacts

- `Category Charter`
- `Research Dossier`
- `Refinement Report`
- `Verification Report`

## Closure Rule

A category is not complete until blockers are closed or logged in the `Exception Ledger`.
