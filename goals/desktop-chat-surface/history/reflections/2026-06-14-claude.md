# Reflection — desktop-chat-surface (2026-06-14, claude)

## Summary

Ported the `effect-lexical-chat` POC into the beep-effect vertical slices and
assembled the desktop chat surface end to end: the agents-slice turn kernel
(Anthropic + deterministic fixture behind one `AgentTurnKernel` interface),
`scanChunk` + its property test, the md-aligned block schema + `@beep/md` lift,
workspace persistence + `ThreadTimeline` (PGlite-proven), `ChatRpcs` + the
`agents-client` atoms (with the `AtomRegistry` interrupt-cleanup lesson),
the app-local orchestration handler + fixture contract tests (cancel-leaves-no-
partial-row, UsageRecord-at-finalization), real UsageRecord persistence +
migration, the chat UI, the live runtime Layer + a bun sidecar that **boots,
serves rpc, and migrates PGlite in this environment**, and the Tauri packaging
(`build-sidecar` bun-compiles; `cargo check` passes). Every CI-verifiable
acceptance criterion is green; the real-LLM E2E + full `tauri build` are the
dev-machine/Anthropic-key remainder the SPEC already delegates to a dev machine.

## Tooling experience

- **Worked:** The vertical-slice archetypes (canvas/server, architecture-lab/server
  for real Drizzle persistence + std-09 error translation) were faithful, copyable
  templates. The PGlite testcontainers harness (`BEEP_TEST_DATABASE_DRIVER=
  pglite-testcontainers`) made persistence genuinely CI-verifiable. `@beep/anthropic`,
  `@beep/md`, `@beep/editor`, `@beep/lexical-schema` exposed exactly the surfaces
  the port needed (`AnthropicTurnPlan`, `Document`, `EditorComposer`/`EditorViewer`,
  `editorStateToDocument`/`documentToEditorState`). Delegating each increment to a
  fresh effect-code-writer with a precise spec + verify gate scaled cleanly.
- **Didn't:** `yeet verify` is the only place the fallow dead-code/audit lane runs,
  so a fully per-package-green feature still failed the final gate on unused
  exports/deps — a signal that arrives too late.
- **Frustrating:** New-package quality artifacts (docgen proof, catalog shard,
  dtslint, JSDoc-that-docgen-compiles) are all generated/hand-authored and only
  fail at the aggregate gates; the scaffold clone doesn't seed them.
- **Wished existed:** A package generator that emits a yeet-clean skeleton, and a
  way to run the fallow lanes scoped to a package mid-development.

## Implementation improvement opportunities

- v1 trimmed mermaid/table/youtube blocks (table is a SPEC non-goal; the others
  have no `@beep/md` node). A follow-on can add a mermaid block as a fenced-code
  (`Pre` lang=`mermaid`) round-trip + a renderer.
- The Anthropic kernel drops structurally-invalid blocks (logged) rather than the
  POC's batched Haiku repair tail — fine for v1; the repair service is a clean
  follow-on if decode-failure rates warrant it.
- UsageRecord on the fixture path hardcodes a `System` principal + `"fixture"`
  provider; the live Anthropic path must thread real provider/model/token/latency
  turn-metadata + the request principal into `appendTurnFinalizationUsageRecord`.
- The version-selector is degenerate (renders only with sibling branches); full
  branch-tree-free version UX is the named follow-on.

## Goal & prompt critique

- The compact `GOAL.md` launcher + layered SPEC/PLAN/manifest contract worked well
  and the "stop before deps/lockfiles/migrations/infra unless SPEC requires" rail
  was the right guard at this scale.
- Two edits would sharpen it: (1) split acceptance into CI-verifiable vs
  dev-machine-proof buckets so an autonomous run has an unambiguous done-state; (2)
  enumerate which persistence substrate exists vs must be built (the epistemic
  `UsageRecord` table/migration was absent and had to be created).

## Lessons (confidence-tiered)

- **HIGH — Critical:** Interrupt cleanup in atoms must write via `AtomRegistry`
  (`registry.set` + `reactivity.invalidateUnsafe`), never `ctx.set` in
  `Effect.onInterrupt` — the node Lifetime is disposed before the fiber unwinds, so
  ctx writes are silently dropped. Ported verbatim from the POC into `runTurnAtom`.
- **HIGH — Critical:** Cancel must persist nothing (assistant turn only on
  `Stream.onEnd`); the POC's partial-persist-on-interrupt contradicts this SPEC.
- **MEDIUM — Best practice:** Provider adaptation (`AnthropicStructuredOutput.
  toCodecAnthropic`) belongs in the server/kernel, not domain (std-04); the pure
  block schema + md lift stay in domain. The codec building at module load doubles
  as a proof the v1 block scope is provider-expressible.
- **MEDIUM — Best practice:** Cross-slice features compose app-locally (std-01
  forbids slice-to-slice imports); ports in use-cases, impls in server, wiring in
  the app's `runtime/Layer.ts`.
- **LOW — Consideration:** Verify each new package's full quality artifacts
  (docgen/catalog/dtslint) as it lands, not at the end, to avoid a long final
  cleanup tail.
