# Reflection - canonical-slice-factory (2026-06-12, codex)

## Summary

This packet successfully replaces the drifted `fixture-lab/Specimen` proof with
the `architecture-lab` WorkItem factory proof and closes V1 around a bounded set
of archetypes. The durable lesson is that the schema-backed operation plan is
the product surface; individual scaffold commands are just ergonomic wrappers.

## Tooling experience

- **Worked:** The packet's source hierarchy, retained automation reference, and
  manifest made it clear that `architecture-operation-plan/v1` is a replay
  contract and that the old fixture proof is reference-only.
- **Didn't:** The packet lacks a single closeout evidence file with commands,
  PR, and CI outcomes, which makes historical verification slower than it needs
  to be.
- **Frustrating:** The handoff prompt is necessarily long because it has to
  guard against rehydrating old fixture topology while still using old code as
  reference material.
- **Wished existed:** A reusable architecture-factory closeout template for
  replay fixture proof, second-apply no-op proof, and stale-reference audits.

## Implementation improvement opportunities

- Keep every future architecture wrapper normalized through the decoded
  operation-plan core before writing files.
- Treat additional domain kinds, foundation-package factories, and driver
  factories as future operation-plan extensions rather than V1 cleanup.
- Record closeout evidence in one artifact when similar architecture packets
  move from implementation proof to retained guidance.

## Goal & prompt critique

The packet is strong because it states both what replaced the old proof and
what V1 deliberately did not absorb. I would revise the launcher to include one
short closure sentence: "V1 is done when WorkItem, Worker, WorkPriority,
db-admin aggregation, and current `beep architecture` plan/apply/check replay
are hardened; do not add new archetype families as closure blockers."

## Lessons (confidence-tiered)

- **HIGH - Critical:** protect versioned operation plans as replay contracts -
  generator ergonomics can change, but existing v1 fixtures must still decode
  and replay identically.
- **MEDIUM - Best practice:** retain deleted proof packets as compact reference
  notes instead of letting old topology remain active guidance.
- **LOW - Consideration:** add a small packet evidence file even when README,
  SPEC, and manifest all agree on closure; it pays for itself during later
  advisory backfills.
