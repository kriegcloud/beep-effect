# Reflection - storybook-app (2026-06-12, codex)

## Summary

This packet cleanly moved Storybook hosting from `@beep/ui` into the private
`@beep/storybook` app while leaving story files package-local. Verification
shows local app, UI, root Storybook, infra, and Pulumi preview surfaces were
proved without mutating live infrastructure.

## Tooling experience

- **Worked:** The `$grill-with-docs` findings became useful guardrails:
  executable app shape, package-local stories, no public API, app-owned browser
  tests, and Vercel-only preview scope.
- **Didn't:** Remote CI was not executed from the local implementation session,
  so the packet correctly records that GitHub should rerun CI on the carrying
  branch or PR.
- **Frustrating:** Storybook touches app config, package story discovery,
  Playwright browser tests, workflow artifacts, and infra wiring at once; a
  narrow source hierarchy is necessary to avoid accidental package API churn.
- **Wished existed:** A standard Storybook-host migration checklist for
  monorepo apps with package-local stories.

## Implementation improvement opportunities

- Keep story discovery intentionally scoped to foundation UI-system story paths
  until shared or slice UI packages deliberately join.
- Preserve the portless service name while letting the owning Turbo filter move
  to the Storybook app.
- Keep Vercel provisioning as a later explicit action; preview proof is enough
  for this packet.

## Goal & prompt critique

The GOAL is precise and correctly short for `/goal` use. I would keep the
negative constraints in the launcher because they prevent the most likely
regression: turning `@beep/storybook` into a library package with exports and
docgen obligations.

## Lessons (confidence-tiered)

- **HIGH - Critical:** a Storybook host can be an executable app without owning
  the stories themselves; keep story authorship with component packages.
- **MEDIUM - Best practice:** preview infrastructure with injected secrets and
  record planned resources, but avoid live mutation unless the spec asks for it.
- **LOW - Consideration:** local runtime proof like `storybook.beep.localhost`
  is useful evidence, but remote CI still needs to rerun on the PR.
