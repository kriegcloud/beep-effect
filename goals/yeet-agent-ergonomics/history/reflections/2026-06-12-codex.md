# Reflection - yeet-agent-ergonomics (2026-06-12, codex)

## Summary

This packet hardened Yeet as an agent-friendly commit-to-mergeable-PR path:
dirty-worktree staged publishes, stale-base refusal, verdict artifacts,
in-flow PR creation, proof-lock recovery, dependency-change cache forcing, and
explicit review-thread write-backs. The closeout evidence is especially strong
because the implementation PR dogfooded the new path.

## Tooling experience

- **Worked:** The packet grounded each enhancement in a real operator session
  and then preserved the dependency order in PLAN, which kept the implementation
  from becoming a grab bag of flags.
- **Didn't:** Broad proof output is still hard to summarize perfectly; the
  known tail-needle hint issue shows verdict data and repair hints need tighter
  coupling.
- **Frustrating:** Staged-only publishing has sharp edges around partially
  staged files, stash restoration, and proof failure paths, but the spec names
  each edge explicitly enough to test.
- **Wished existed:** A structured Yeet run timeline viewer that combines
  verdict, packet, PR body, closeout report, and stash/base freshness state.

## Implementation improvement opportunities

- Improve sub-lane hint selection by tying needles to the failed lane output
  instead of the final broad proof tail.
- Keep `--allow-stale-base` and write-back flags explicit so speed and
  convenience do not weaken operator intent.
- Continue exporting pure helpers as `*ForTesting` when behavior is subtle and
  needs focused coverage.

## Goal & prompt critique

The goal is unusually strong because it requires dogfooding and concrete hosted
closeout outcomes, not just local tests. I would keep the "one commit per
implementation phase" constraint for similar operator packets because it made
review and rollback easier.

## Lessons (confidence-tiered)

- **HIGH - Critical:** agent automation should write machine-readable verdicts
  on success and failure; otherwise every interrupted proof becomes archaeology.
- **MEDIUM - Best practice:** add convenience flags only with explicit refusal
  states, packet artifacts, and tests for dirty or stale worktrees.
- **LOW - Consideration:** closeout write-backs are powerful, but keeping them
  one-thread-at-a-time and opt-in preserves review trust.
