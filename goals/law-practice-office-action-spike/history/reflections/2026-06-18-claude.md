# Reflection — law-practice-office-action-spike (2026-06-18, claude)

## Summary

The spike landed end-to-end and green: a clean `law-practice` slice (domain →
use-cases → server) maps span-bearing extraction output into IP-law entities,
drives a single distinction-claim through the epistemic `ClaimGate` →
`ClaimTransition` → `ClaimProjection` loop, and answers the one trivial ask
("what distinguishes claim 1 over the cited reference, and where is the
evidence?"). Global `bun run check` is EXIT=0; the domain tier imports only
foundation + shared-kernel, with the cross-slice epistemic composition confined
to the use-cases/server tiers per the documented Exception Ledger entry.

## Tooling experience

- **Worked:** `bunx turbo run check --filter=…` gave fast per-package
  iteration only; the authoritative proof stayed the global repo check;
  the existing P0 test and `EpistemicUseCases.test.ts` were precise templates for
  entity decode shapes and the gate/transition/projection composition;
  `bun run beep docgen init -p <pkg>` cleanly produced the docgen config once I
  found it; `it.layer(LawPracticeServerLive)` made the integration test a direct
  expression of the loop.
- **Didn't:** `config-sync` silently no-op'd for unregistered packages (no error,
  no aliases) — twice. The `Read` tool's "wasted call" guard repeatedly blocked
  re-reads of files that a mid-session merge may have changed, forcing `cat`
  fallbacks.
- **Frustrating:** the `schemaSyncInEffect` and `effectFnOpportunity` tsgo lints
  only surface under `check`, not at authoring time, so the first impls compiled
  in my head but failed the gate; the docgen internal-example failure took two
  wrong turns (package path → relative path) before the real fix (exclude glob).
- **Wished existed:** an authoring-time signal for the Effect-LSP lints; a
  `config-sync` warning for orphaned packages; a documented "new package
  checklist" (workspaces entry → install → config-sync → docgen init).

## Implementation improvement opportunities

- The `spikeEntityInput` shim duplicates `@beep/test-utils` `baseEntityFixtureInput`
  because the latter is a devDep unimportable from `src`. A shared test-only
  fixtures entrypoint would remove the duplication and the spike caveat.
- The IrToLaw mapper currently hard-codes fixture keys and a §102 ground; when a
  second statute or multi-reference §103 is needed, the label→entity mapping
  should move to a small typed table rather than inline `findLabel` calls.
- The fixed candidate set lives both in the loop impl and (mirrored) in the test;
  exposing the candidate set from the use-cases package would keep them from
  drifting, at the cost of widening the public surface — deferred deliberately.

## Goal & prompt critique

The GOAL/SPEC were unusually well-scoped — the Exception Ledger, the
matter-wall invariant, and the "trivial ask only" boundary all prevented scope
creep, and the per-phase briefs were precise. Two edits would have saved an
iteration each: (1) the SPEC should have named `GroundedExtraction` (not the nlp
`AnnotatedDocument` envelope) as the IR→law input from the start, since the
envelope is span-lossy at the entity level; (2) the SPEC should state explicitly
that gate-admission lands the claim at `shape_valid` with empty `admittedKeys`,
which is counter to the natural reading of "admitted."

## Lessons (confidence-tiered)

- **HIGH — Critical:** When a downstream target requires a source anchor, the IR
  it consumes must itself carry spans — verify the producer's envelope actually
  ships span-bearing records before designing the contract against it. The nlp
  `AnnotatedDocument` envelope does not, and the P1 contract had to be retargeted.
- **HIGH — Critical:** Decode schemas at module scope or with `decodeUnknownEffect`,
  never `decodeUnknownSync` inside an Effect generator/thunk — the tsgo
  `schemaSyncInEffect` lint fails `bun run check` otherwise.
- **HIGH — Best practice:** A new workspace package is invisible to `config-sync`
  until its path is in the root `package.json` `workspaces` array; add it, then
  `bun install`, then `config-sync`, then `docgen init`.
- **MEDIUM — Best practice:** Compose layers so a provider and a consumer of the
  same tag never sit in the same `Layer.mergeAll`; use
  `consumer.pipe(Layer.provideMerge(provider), Layer.provide(rest))` to satisfy
  the `layerMergeAllWithDependencies` lint while keeping the provider in the
  output surface.
- **MEDIUM — Consideration:** "Gate admits" ≠ lifecycle `admitted`; an admitted
  verdict advances `candidate → shape_valid` by one step, so a freshly-reviewed
  claim shows `counts.shape_valid===1` and `admittedKeys===[]`.
