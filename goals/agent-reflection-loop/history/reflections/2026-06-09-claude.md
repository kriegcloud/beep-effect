# Reflection — agent-reflection-loop (2026-06-09, claude)

## Summary

P1 landed a working schema-first reflection capability: the `ReflectionFrontmatter`
model + `bun run beep lint reflection-artifacts` advisory/enforcement rule + the
`/reflect` skill + the `_template` topology. The rule routes, runs, and reports
`blocking_findings=0` with 4 legacy advisories. This file is the dogfood proof.

## Tooling experience

- **Worked:** `SchemaFirst.ts` was an excellent template — `LiteralKit`, `S.Class`,
  the `[*:issue]` JSON-line encoding (`encodeUnknownEffect` + `stringifyJsonLine.run`),
  and `failWithReportedExit` transferred almost directly. `decodeYamlTextAs` made
  frontmatter validation a one-liner.
- **Didn't:** the lint-subcommand routing gate (see findings) cost the most time.
- **Frustrating:** `bun run beep` cold-start latency made each rule run slow to verify.
- **Wished existed:** a single source of truth for "is this a CLI lint policy subcommand".

## Implementation improvement opportunities

- Inventory-baseline option for finer grandfathering (see findings).
- The rule re-reads every goal manifest each run; fine at this scale, but a future
  `--changed` mode would help in large monorepos.

## Goal & prompt critique

The goal ("full schema-first now" + "dedicated packet") was clear and the
opt-in `reflectionRequired` flag made "enforcement from day one" non-breaking. The
only revision I'd request: surface the command-routing gate in the prompt.

## Lessons (confidence-tiered)

- **HIGH — Critical:** registering a `beep lint` subcommand requires updating
  `Lint.command.ts` + `bin-main.ts` `LINT_POLICY_SUBCOMMANDS` + `Tasks.ts`
  `LintPolicySubcommand` (+ a composite step) — or it silently routes to turbo.
- **MEDIUM — Best practice:** mirror an existing self-contained rule
  (`SchemaFirst.ts`) wholesale rather than splitting schema/rule across files;
  it minimizes new public exports and jsdoc burden.
- **LOW — Consideration:** `Effect.orElseSucceed(A.empty<string>)` (direct helper
  ref) satisfies the terse-effect law where `() => A.empty<string>()` does not.
