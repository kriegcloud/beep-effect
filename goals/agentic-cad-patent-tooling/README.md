# Agentic CAD — Patent Tooling

Tool-discovery / buyer's-guide research on AI/LLM-driven CAD tooling for a solo IP/patent
attorney. The report evaluates each tool on its own published merits and integration surfaces —
it is deliberately **repo-agnostic** and does not assume or design any downstream integration
architecture.

The research covers four jobs-to-be-done: (1) USPTO/WIPO patent **figures**, (2) invention
**3D modeling**, (3) **prior-art / reverse-modeling**, and (4) the full **disclosure→figure
pipeline**. Each tool is scored on two axes — an *integration surface* describing what the tool
itself exposes (native MCP → REST/gRPC API → CLI/headless → scriptable library → file-format
interop), and a *deployment / confidentiality* axis (cloud / hybrid / self-hostable / fully-local,
with attorney-confidentiality / Rule 1.6 implications for privileged matter data).

## Status

**Research complete (2026-05-29).** Headline: adopt the local OpenCASCADE Python stack —
**`CadQuery` + `build123d`** — first. Both are Apache-2.0 and run fully offline (privilege-safe
under Rule 1.6), and together they cover three of the four jobs; `build123d` uniquely projects 3D
models to patent black-line 2D figures (`project_to_viewport` / `TechnicalDrawing` /
`ExportSVG`·`ExportDXF`). Cloud text-to-CAD (adam.new, Zoo, Autodesk neural CAD, MakeIt3D) is an
opt-in accelerator for non-privileged / already-public work only — Zoo's free tier trains on user
data (a Rule 1.6 red flag). No turnkey disclosure→figure pipeline (Job 4) exists; it is
assemble-it-yourself.
