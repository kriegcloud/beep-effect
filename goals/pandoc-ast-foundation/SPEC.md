# Pandoc AST Foundation Spec

## Objective

Create a pure foundation/modeling package, `@beep/pandoc-ast`, that can decode
and encode committed Pandoc JSON fixtures, map the supported Pandoc Md-core
profile to and from `@beep/md`, and report unsupported DOCX-origin constructs
without invoking a live Pandoc executable.

## Non-Goals

- No Pandoc process driver, sidecar, CLI wrapper, or Tauri integration.
- No generated DOCX fixture pipeline.
- No editor UI or user-facing document workflow.
- No patent-specific semantic nodes or legal-domain extraction.
- No semantic changes to `@beep/md`.

## Source Hierarchy

1. User objective: implement the graduated `docx-roundtrip-interop` plan.
2. `AGENTS.md`, `CLAUDE.md`, and required skills.
3. `explorations/docx-roundtrip-interop/{DECISIONS.md,MAP.md}`.
4. Repo architecture/package standards and schema-first doctrine.
5. This `SPEC.md`.
6. `PLAN.md`.
7. `GOAL.md`.

Higher sources outrank lower sources when they conflict.

## Target Surfaces

- `packages/foundation/modeling/pandoc-ast`
- package registration/linkage generated by `bun run beep create-package`
- `goals/pandoc-ast-foundation`
- `explorations/docx-roundtrip-interop`
- `explorations/ATLAS.md`

## Constraints

- Keep normal tests independent of a local `pandoc` executable.
- Use schema-first domain models and typed compatibility report data.
- Public package imports must work through `@beep/pandoc-ast` aliases.
- Preserve DOCX-origin gaps as data; do not hide them behind string logs.
- Keep v1 mapping bounded to Md-core plus explicit lossiness reports.
- Avoid broad repo refactors and unrelated formatting churn.

## Acceptance Criteria

- [x] `@beep/pandoc-ast` exposes root and explicit subpath exports for
      `Pandoc.model`, `Pandoc.codec`, `Pandoc.mapping`, and `Pandoc.report`.
- [x] Committed Pandoc JSON fixtures decode through schema-owned JSON codecs.
- [x] Supported Md-core fixture maps to `@beep/md` with `profile: "supported"`
      and no issues.
- [x] DOCX-style fixture records structured issues for custom style wrappers,
      notes, math, and tables while producing partial Md output.
- [x] `@beep/md` to Pandoc mapping reports raw Markdown/HTML and task-list
      lossiness.
- [x] Package-local check, test, type-test, lint, docgen, and build pass.
- [x] Exploration docs/manifests show graduation into this packet.
- [x] No unrelated refactors or formatting churn.
