# Reflection — epistemic-claim-lifecycle-gate (2026-06-17, claude)

## Summary

Graduated the epistemic slice from domain-only to minimum-viable (domain +
use-cases + server): a four-state `ClaimLifecycle` machine, the ported v3
`EvidenceSpan` char-offset primitive, a `ClaimGate` composing the bounded SHACL
engine, a pure rebuildable `ClaimProjection`, and a thin `EpistemicServerLive`
Layer. Built strictly in phase order (P0 schema → P1 contract → P2 impl → P3
verify); all four tiers type-check, lint clean, and 11 epistemic tests pass with
zero IP-law vocabulary. Verdict: complete and on-spec.

## Tooling experience

- **Worked:** The parallel "understand" workflow (6 readers) front-loaded the
  exact SHACL/RDF API, slice scaffolding, and standards with citations — P0–P2
  proceeded with almost no exploratory backtracking. `bun run config-sync`
  (`beep tsconfig-sync`) is excellent: registering two new packages auto-wired
  root tsconfig aliases, syncpack sources, and quality references with zero drift.
  `S.decodeUnknownSync` from plain objects made constructing branded/tagged-union
  values painless.
- **Didn't:** `tsgo` isn't on `PATH` for the literal verification command
  `tsc --noEmit -p .../use-cases/tsconfig.json`; the working path is
  `bun run check --filter` (turbo → `tsgo -b`).
- **Frustrating:** The persist `StorageKind` set has no float, and that constraint
  only surfaces by reading `EntitySchema.persist.ts` + `BaseEntity.ts` — it
  silently dictates the whole Evidence data model. The `export type` duplicate on
  `S.Class` exports is an easy, repeated footgun.
- **Wished existed:** A foundation 0..1 `Confidence` primitive; a documented
  "fractional scalar → jsonb value object" persistence recipe.

## Implementation improvement opportunities

- Embed `EvidenceSpan` as a jsonb value object (done) — but once `@beep/provenance`
  lands, refactor it to wrap a foundation `TextAnchor` so the four "where-from?"
  shapes converge (cross-linked in `EvidenceSpan` JSDoc).
- `ClaimGate.evaluate` uses `Effect.orDie` on the bounded engine's (never-fired)
  error channel to keep the gate's failure surface a value-only verdict. If a real
  external SHACL engine later backs the contract, revisit so `invalidShape` /
  `engineFailure` map to a typed outcome rather than a defect.
- The gate's SHACL shape currently asserts "≥1 evidence quote". When the claim
  shape needs richer constraints than `targetClass`/`minCount`/`maxCount`/`datatype`,
  add Effect-level checks rather than stretching the bounded engine.

## Goal & prompt critique

- The `.ports.ts`/`.service.ts` role vocabulary diverges from the only reference
  slice (`architecture-lab` uses `.repository.ts`). Either align the SPEC to the
  repo convention or call out the intended divergence so the next agent doesn't
  second-guess it.
- GOAL.md implies char offsets replace the fixture-key refs; it actually adds them
  alongside. One clarifying clause would remove the ambiguity.
- Otherwise the packet was unusually executable: binding phase gates, explicit
  HAVE/NET-NEW surface table, and a normative SPEC made the work mechanical.

## TODOs worth codifying

- Foundation `UnitInterval`/`Confidence` primitive in `@beep/schema/Number`.
- Numeric/real `StorageKind` or a documented jsonb-value-object recipe for
  fractional scalars.
- Schema-first rule: no `export type X = typeof X.Type` after `S.Class`/`BaseEntity.Class`.
- `@beep/provenance` follow-on to unify evidence anchoring.

## Lessons (confidence-tiered)

- **HIGH — Critical:** Check `StorageKind` + `BaseEntity` persist requirements
  BEFORE designing any entity field — the absence of a float kind forced a
  value-object-as-jsonb design that flat fields could never satisfy.
- **HIGH — Critical:** For `S.Class`/`BaseEntity.Class` exports, the class is the
  type — never add `export type X = typeof X.Type`; do add it for `const`/LiteralKit schemas.
- **MEDIUM — Best practice:** Build annotated tagged-union / branded values with
  `S.decodeUnknownSync(Schema)(plain)` rather than depending on `.cases`/`.make`
  surviving `$I.annoteSchema`.
- **MEDIUM — Best practice:** Keep use-cases layer-free — a pure
  `makeX(resolvedDep)` factory plus a server-tier `Layer.effect` that resolves the
  capability keeps the boundary honest (gate depends on `ShaclValidationService`'s
  type; only the server provides its Live layer).
- **LOW — Consideration:** When a packet prescribes file-role suffixes, confirm
  them against the reference slice before scaffolding.
