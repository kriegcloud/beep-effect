# Gap: adjunct vs current `@beep/nlp`

> **Merge target** `@beep/nlp` (Effect v4, foundation/capability) is already populated (50 src + 7 tests) and on v4 idioms (`$NlpId` identity, `TaggedErrorClass`, `Clock`, `Context.Service`, `effect/unstable/ai`). The adjunct source (`@template/basic`, Effect v3.17.7) overlaps `@beep/nlp` in several areas and is genuinely new in others. Disposition legend: **PORT NEW** (no `@beep/nlp` equivalent), **MERGE** (fold missing parts into `@beep/nlp`), **KEEP @beep/nlp** (target supersedes adjunct; nothing to port), **SUPERSEDE** (drop adjunct in favor of existing `@beep/nlp`).

## `@beep/nlp`-unique — MUST preserve (no adjunct equivalent)

- **Brand/domain text helpers** (not in adjunct): `IdentifierText.ts` (`tokens`, `variants` — camel/Pascal/snake/kebab/joined), `PathText.ts` (`normalizePathPhrase`, `isPathLike`, `filePathVariants`, `moduleSpecifierVariants`), `QueryText.ts` (`normalizeQuestion`, `normalizePhrase`, `extractBacktickValue`), `VariantText.ts` (`orderedDedupe`).
- **`$NlpId` IdentityComposer discipline** — every schema/error/service keyed via `$NlpId.create("module.path")` + `$I\`Name\`` + `$I.annote`/`annoteSchema` (Document, Token, Sentence, Pattern, WinkEngine, all `Ai*` schemas). adjunct uses bare-string identifiers.
- **`TaggedErrorClass` taxonomy** (`@beep/schema`, `cause: S.DefectWithStack`, `fromCause`/`fromMessage`): `TokenizationError`, `WinkEngineError`, `WinkTokenizationError`, `WinkEntityError`, `SimilarityError`, `VectorizerError`, `CorpusManagerError`, `WinkUtilsError`, `ExportedToolError`, + `S.TaggedClass` `SentenceSpanFailure`.
- **Already-on-v4 toolkit** — `NlpToolkit`/`NlpToolkitLive`/`NlpTools` + 19 `Tool.make` defs + `Ai*` success schemas (`AiToken/AiSentence/AiKeyword/AiEntity/AiNGram/AiPhoneticMatch/AiDocumentStats/AiSentenceChunk/AiRankedText` + corpus `Ai*`) on `effect/unstable/ai`; `ToolExport` via `Tool.getJsonSchema`/`getDescription`. **This is the v4 target adjunct's `@effect/ai` + MCP SDK code must migrate to.**
- **Clock-correct, Ref-based stateful services** — `WinkEngine`, `WinkVectorizer`, `WinkCorpusManager` read time via `Clock.currentTimeMillis`, hold state in `Ref`/`HashMap`/`HashSet` (no wall-clock, no native Map/Set except the WONTFIX wink-FFI Set).
- **Decomposed service seam** — `Core/Tokenization` (backend-agnostic `Context.Service` + free `Effect.fn` accessors) separate from `WinkTokenizer` (`Layer.effect` impl). Canonical contract-vs-implementation split.
- **Curated published subpath exports** (`package.json`): `./Core`, `./Core/*`, `./Wink`, `./Wink/*`, `./Tools`, `./Tools/*`, `./IdentifierText`, `./PathText`, `./QueryText`, `./VariantText`, `./Layers`, `./Layers/*`, with `./internal/*` nulled (`internal/numbers.ts` `UnitInterval`/`PositiveNumber`, `internal/order.ts` `descendingNumber`/`ascendingNumber`/`ascendingString`).
- **19 NLP Tools** (preserve as the canonical surface): `Tokenize`, `Sentences`, `ExtractEntities`, `ExtractKeywords`, `DocumentStats`, `ChunkBySentences`, `TransformText`, `NGrams`, `PhoneticMatch`, `TextSimilarity`, `BowCosineSimilarity`, `TverskySimilarity`, `RankByRelevance`, `CreateCorpus`, `LearnCorpus`, `QueryCorpus`, `CorpusStats`, `DeleteCorpus`, `LearnCustomEntities`.
- **7 existing v4 test suites** — already green on v4; do not regress.

## Ordered "What to Build" (staging port — P1)

1. **Resolve the typeclass-algebra design decision FIRST** (gates Algebra/TypeClass/Composable/NLPMonoids): adopt `effect/Reducer` (Monoid) + `effect/Combiner` (Semigroup) and a small LOCAL `Monoid<A>` interface for `combineMany`/`combineAll`; drop Foldable/Traversable/SemiApplicative/Applicative. Fold genuinely-new NLP monoid instances into `@beep`'s pattern algebra. Port the monoid law suites.
2. **Port the graph node schemas** (`src/Schema.ts` → new `@beep/nlp` graph node classes: `TextNode`, `TextEdge`, `POSNode`, `EntityNode`, `LemmaNode`, `DependencyNode`, `RelationNode`, `NLPAnalysis`) with `$NlpId` ids, annotations, and Clock timestamps — the handoff-contract basis for everything graph.
3. **PORT NEW: Graph subsystem** (`@beep/nlp/Graph`) on built-in `effect/Graph`: `EffectGraph` (capture `NodeIndex` from `Graph.addNode`; `makeNode` Clock-effectful), `TextGraph` (`addChildren` Effect-failing on cycle), `AnnotatedTextGraph`, `GraphOps` (reconstruction-based `mapNodes<A→B>`/`bimap`/`merge`; `SearchIndex` over `HashMap`), `GraphOperations` services (`GraphExecutor`, `ResultStore`, `Catalog`, `Operation`, `Types`, `Errors`, `Schemas`) as `Context.Service` + `Layer`. Port cata/ana + graph/adjunction/functor law suites.
4. **PORT NEW: Ontology/Kind** (`@beep/nlp/Ontology`) — TypedText Kind union + containment poset with `$NlpId` + `LiteralKit`; port the poset property tests.
5. **MERGE: NLP service ops** — diff adjunct `NLPService` against `Core/Tokenization` + wink services; fold any missing operations (e.g. paragraphize/normalize variants, stringSimilarity if absent) into existing seams. Port NLP-operation law suites onto `@beep` services (single-runtime `it.layer` form, no nested `runPromise`).
6. **PORT NEW (conditional): Streaming subsystem** — if streaming corpus ingestion is required: `TextStream`, `Jsonl` (via `effect/unstable/encoding/Ndjson`), `DatasetLoader` (via `effect/unstable/http/HttpClient`, fix missing-import bug), `Cache` (via `effect/Cache`), `Pipeline` (direct Stream), `StreamingTools` + `StreamingHandlers` as `effect/unstable/ai` Tools/handlers.
7. **PORT NEW (conditional): multi-backend Composition** — only if pluggable backends are a goal: `FallbackBackend`/`CachingBackend`/`selectBackend` over `@beep`'s wink services + a backend `Context.Service` abstraction; `safeCall`/`tryOperation` returning `Option`.
8. **EVALUATE: Formatter** — only if display output is needed; reimplement without `@effect/printer`/`-ansi` (hand-rolled ANSI or plain strings), or rely on `@beep` `toFormatter` annotations.
9. **Drop / do not port:** `Operations/Registry.ts` (@deprecated), `Mcp/{Server,Tools,Schemas}` (superseded by `NlpToolkit`/`ToolExport`), `zod`/`@modelcontextprotocol/sdk`/`@effect/ai*`/`@effect/printer*`/`@effect-atom/atom-react`, `src/Program.ts`/`src/AdvancedDemo.ts` (demos), `web/`, `EffectGraph.ts.old`.
