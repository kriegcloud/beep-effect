# Reflection — provenance-shared-claim-kernel (2026-06-18, claude)

## Summary

Resolved the cross-slice collision that blocked `law-practice` from consuming the
`epistemic` boundary, by routing the coupling to correct homes: domain-agnostic
substrate (`UnitInterval` → `@beep/schema`; char-offset anchor → new
`@beep/provenance` `TextAnchor`) to `foundation/modeling`; the minimum product
vocabulary (`ClaimLifecycle`) to `shared/domain` with a promotion record; the
mechanism (gate/projection services + live Layer) left in the `epistemic` slice.
An adversarial review confirmed the routing is clean and surfaced three real P1
issues (all fixed). Verdict: the foundation is sound and ready for the consumer
spike to build on.

## Tooling experience

- **Worked:** mirroring an existing peer (`Percentage.ts` for the primitive, `rdf`
  for the new package) made scaffolding fast and idiomatic; `config-sync` cleanly
  registered the new package + subpath aliases from the exports map; the
  Workflow adversarial-review pattern was high-leverage.
- **Didn't:** per-package `bunx turbo run check --filter` gave false confidence —
  it skips the dtslint/tstyche lane, so a tracked type-test was red while I
  reported green.
- **Frustrating:** the new-package scaffold is ~11 hand-authored files
  (package.json dev+publish exports, two tsconfigs, vitest config, README,
  AGENTS, CLAUDE symlink, LICENSE, docgen.json, barrel) — a generator would
  remove a lot of copy-and-adjust.
- **Wished existed:** a single "is this change non-breaking?" command that runs
  tsgo + tstyche + the public-surface diff for the touched packages.

## Implementation improvement opportunities

- `TextAnchor` encodes its `startChar<=endChar` invariant as a predicate
  (`isWellOrdered`) rather than a schema filter, because no modeling package uses
  a cross-field `S.filter` idiom yet. Once one is adopted, bake the invariant in.
- `Confidence` is aliased directly to `UnitInterval`, so decode errors say
  "UnitInterval" not "Confidence". Acceptable, but a re-branded alias would keep
  the epistemic-semantic error identity if that matters later.
- The `@beep/nlp` internal `UnitInterval` duplicate should converge on the new
  foundation primitive.

## Goal & prompt critique

The GOAL/SPEC were derived live from the grilling session and were accurate. One
edit worth making for future similar packets: the GOAL should state explicitly
that "non-breaking" must be proven with the dtslint/tstyche lane, not just
per-package `check`, since type-level regressions are exactly the breakage a
re-point risks. The packet correctly scoped the consumer (`law-practice`) OUT —
that separation held up well.

## TODOs worth codifying

- A non-breaking-change verification recipe (tsgo + tstyche + export-surface diff)
  promoted into the `yeet`/quality tooling.
- `lint:promotion-records` (already "planned" in doctrine) to mechanically reject
  promotion records that name non-importing consumers.
- New-foundation-package generator to replace the 11-file hand scaffold.

## Lessons (confidence-tiered)

- **HIGH — Critical:** "Green" from per-package `turbo check` + `test` is not
  "green" — the dtslint/tstyche and Effect-LSP lanes only run under full
  `bun run check`. Prove non-breaking type claims against the type-test lane.
- **HIGH — Critical:** A shared-kernel promotion needs two REAL current importers.
  Promoting ahead of the second consumer is allowed only as a documented,
  provisional decision with a removal trigger — never by asserting an unbuilt
  consumer as present fact.
- **MEDIUM — Best practice:** Adversarially review a new shared/foundation surface
  before consumers build on it; independent refutation catches what uniform green
  gates miss, and confirms the wins (here, the clean domain-tier routing).
- **LOW — Consideration:** Routing a cross-slice coupling by tier
  (substrate→foundation, language→shared-kernel, mechanism→slice) is a reusable
  pattern; it is now recorded as doctrine in `DECISIONS.md` for the next vertical.
