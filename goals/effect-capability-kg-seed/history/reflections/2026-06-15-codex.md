# Reflection - effect-capability-kg-seed (2026-06-15, codex)

## Summary

This packet landed the first deterministic Effect capability KG seed in
`@beep/repo-utils`: it extracts Effect v4 AST/JSDoc/catalog evidence for
`Combiner`, `Reducer`, `Filter`, adjacent helpers, graph edges, and advisory
fixtures. The outcome is ready as a seed proof, with hooks, embeddings, storage,
and enforcement still intentionally out of scope.

## Tooling experience

- **Worked:** `TSMorphService`, schema-first `S.Class` models, `LiteralKit`, the
  repo export catalog shard workflow, and Effect v4's standardized JSDoc gave
  this packet a strong deterministic spine.
- **Didn't:** `bunx turbo run lint --filter=@beep/repo-utils` was blocked by an
  unrelated formatter issue in `@beep/identity/src/packages.ts`, so scoped
  package lint had to be recorded separately.
- **Frustrating:** The root catalog aggregate had to be refreshed after the
  package shard, and the exact narrow sequence was not obvious until the check
  failed.
- **Wished existed:** A single command for "refresh this package shard, refresh
  aggregate from shards, then check" would reduce generated-file anxiety.

## Implementation improvement opportunities

- Add focused query helpers over `EffectCapabilitySeedReport`, such as
  `findCapabilitySymbols`, `findModuleEdges`, and `findAdvisoryEvidence`, before
  building a router or hook adapter on top.
- Consider exporting selected graph node schemas only when the next consumer
  needs them. For this seed proof, the report class is enough.
- Make advisory scoring scenario-aware rather than mostly keyword-token based
  once more fixtures are added.

## Goal & prompt critique

The packet did a good job keeping hooks, embeddings, storage, and hard
enforcement out of scope. I would revise the verification matrix to add this
line for public tooling exports:

- Run package docgen and package-local lint when public APIs are created, and
  record any Turbo dependency-lane failures separately with file and command
  evidence.

## Lessons (confidence-tiered)

- **HIGH - Critical:** Keep the deterministic fact layer authoritative before
  ontology, embeddings, or agent routing. The useful facts here came from AST,
  JSDoc tags, source spans, import edges, examples, and repo catalog entries.
- **MEDIUM - Best practice:** When adding a public wildcard subpath, refresh the
  package shard, refresh the root aggregate from shards, and run
  `bun run repo-exports:catalog:check`.
- **LOW - Consideration:** A tiny query facade over the report will likely save
  work in the next packet, but it should wait until one real router or hook
  consumer exists.
