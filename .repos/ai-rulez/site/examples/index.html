    Examples

    Start with the simplest setup and progressively build more complex configurations.

    1. Minimal Setup

    The absolute simplest configuration - one file, one rule:

    metadata:
  name: "My Project"

rules:
  - name: "code-quality"
    content: "Write clean, readable code"

outputs:
  - path: "CLAUDE.md"

    ai-rulez generate  # Creates CLAUDE.md

    2. Add Multiple Rules

    Add priorities to control ordering:

    metadata:
  name: "My Project"

rules:
  - name: "typescript-strict"
    content: "Use TypeScript strict mode with explicit types" 
    priority: 10

  - name: "testing"
    content: "Write unit tests for all functions"
    priority: 8

outputs:
  - path: "CLAUDE.md"

    3. Multiple AI Platforms

    Generate files for different AI tools:

    metadata:
  name: "My Project"

rules:
  - name: "code-quality"
    content: "Write clean, readable code"
    priority: 10

outputs:
  - path: "CLAUDE.md"
  - path: ".cursor/rules/rules.mdc"
  - path: ".github/copilot-instructions.md"

    Or use presets for quick setup: 

    ai-rulez init "My Project" --popular  # Claude, Cursor, Windsurf, Copilot

    4. Add Documentation Sections

    Include longer documentation blocks:

    metadata:
  name: "TypeScript App"

rules:
  - name: "typescript-strict"
    content: "Use TypeScript strict mode with explicit types"
    priority: 10

sections:
  - title: "Project Setup"
    content: |
      This project uses:
      - TypeScript 5.0+
      - React 18
      - Vite for bundling
    priority: 10

outputs:
  - path: "CLAUDE.md"

    5. Claude Agents

    Add specialized sub-agents for specific tasks:

    metadata:
  name: "Enterprise App"

rules:
  - name: "code-quality"
    content: "Write clean, maintainable code"
    priority: 10

agents:
  - name: "code-reviewer"
    description: "Reviews code for best practices"
    system_prompt: "You are a senior code reviewer focused on quality and security"
    tools: ["read", "grep", "edit"]
    priority: 10

outputs:
  - path: "CLAUDE.md"
  - path: ".claude/agents/"
    type: "agent"
    naming_scheme: "{name}.md"

    6. Remote Configuration Sharing

    Share common rules across projects:

    includes:
  - "https://raw.githubusercontent.com/myorg/standards/main/backend.yaml"

metadata:
  name: "User Service"

rules:
  - name: "service-specific"
    content: "Handle user authentication with JWT"
    priority: 10

outputs:
  - path: "CLAUDE.md"

    Remote rules are merged with local ones. Local rules override remote rules with the same name.

    Language-Specific Examples

    Build on the foundation above with these complete project examples.

    Python Data Science Project

    metadata:
  name: "ML Pipeline"

rules:
  - name: "python-version"
    content: "Use Python 3.11+ with type hints"
    priority: 10

  - name: "data-processing" 
    content: "Use pandas for data, numpy for numerical computing"
    priority: 9

  - name: "testing"
    content: "Write tests with pytest, use fixtures for data setup"
    priority: 8

sections:
  - title: "Development Environment"
    content: |
      - Use conda for environment management
      - Pin dependency versions in requirements.txt
      - Use Jupyter notebooks for experimentation only
    priority: 10

outputs:
  - path: "CLAUDE.md"
  - path: ".cursor/rules/rules.mdc"

    React Frontend Application

    metadata:
  name: "React Dashboard"

rules:
  - name: "react-patterns"
    content: "Use functional components with hooks, implement error boundaries"
    priority: 10

  - name: "styling"
    content: "Use Tailwind CSS with design system tokens"
    priority: 9

sections:
  - title: "Project Structure"
    content: |
      ```
      src/
      ├── components/     # Reusable UI components  
      ├── pages/         # Page components
      ├── hooks/         # Custom hooks
      └── services/      # API calls
      ```
    priority: 10

outputs:
  - path: "CLAUDE.md"
  - path: ".cursor/rules/rules.mdc"
  - path: ".windsurf/rules.md"

    Go CLI Tool

    metadata:
  name: "CLI Tool"

rules:
  - name: "cli-design"
    content: "Use cobra for commands, provide helpful error messages"
    priority: 10

  - name: "user-experience"
    content: "Add progress bars, use colors with --no-color flag"
    priority: 9

outputs:
  - path: "CLAUDE.md"
  - path: "AGENTS.md"  # For AMP/Codex

    Start with the minimal examples above, then customize for your specific needs.
