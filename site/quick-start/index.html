    Quick Start

    Get started in 3 simple steps, then expand your configuration.

    1. Create Your First Configuration

    ai-rulez init "My Project"

    This creates ai_rulez.yaml with a minimal Claude setup:

    metadata:
  name: "My Project"

rules:
  - name: "code-quality"
    content: "Write clean, readable code"
    priority: 10

outputs:
  - path: "CLAUDE.md"

    2. Generate AI Files

    ai-rulez generate

    Creates CLAUDE.md with your rules.

    3. Validate Configuration

    ai-rulez validate

    Expanding Your Setup

    Add More Rules

    Edit ai_rulez.yaml:

    metadata:
  name: "My Project"

rules:
  - name: "code-quality"
    content: "Write clean, readable code"
    priority: 10

  - name: "testing"
    content: "Write unit tests for all functions"
    priority: 9

outputs:
  - path: "CLAUDE.md"

    Add Multiple AI Platforms

    # Add popular platforms
ai-rulez init "My Project" --popular

    Or add specific platforms:

    outputs:
  - path: "CLAUDE.md"
  - path: ".cursor/rules/rules.mdc"
  - path: ".github/copilot-instructions.md"

    Use CLI Commands

    # Add a new rule
ai-rulez add rule "error-handling" --content "Handle all errors explicitly" --priority 8

# Add an output  
ai-rulez add output --path ".windsurf/rules.md"

    Advanced: Add Agents (Claude)

    For specialized sub-tasks:

    ai-rulez add agent "code-reviewer" \
  --description "Reviews code quality" \
  --system-prompt "You are a senior code reviewer"

    Git Hooks (Optional)

    Automatically validate configuration on git commit:

    ai-rulez init "My Project" --setup-hooks

    Works with lefthook, pre-commit, or husky.
