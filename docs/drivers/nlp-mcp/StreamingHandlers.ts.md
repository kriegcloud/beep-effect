## StreamingHandlers.ts overview

Handler layer for the streaming MCP toolkit.

Binds each of the 17 `StreamingTools.StreamingToolkit` tools to an
Effect-based implementation backed by the `TextStream`, `Jsonl`,
`DatasetLoader`, and `Pipeline` helpers. Handler failures are
mapped to `AiToolError` (the toolkit declares `failureMode: "return"`),
and spans are annotated with counts, paths, and byte sizes only -- never raw
file content or line/record text.

The resulting layer requires the `FileSystem.FileSystem` and
`Path.Path` services, which are provided at the entrypoint.

Since v0.0.0

# layers

## StreamingToolkitHandlersLive

Live handler layer for the streaming toolkit.

Provide this layer (together with a node `FileSystem`/`Path` implementation)
when mounting `StreamingTools.StreamingToolkit` into an MCP server. Each
handler returns plain objects matching the tool's schema-backed success shape
and surfaces expected failures as `AiToolError`.

**Example**

```ts
import { StreamingToolkitHandlersLive } from "@beep/nlp-mcp/StreamingHandlers"

console.log(StreamingToolkitHandlersLive)
```
