## White-Paper Section Mapping

| White-Paper Section | Primary Source Docs | Supporting Docs |
|---|---|---|
| Executive summary | D02 | D01, D12 |
| Problem and context | D02, D03 | D11 |
| System architecture | D05 | D06, D08, D09 |
| Methodology | D06, D07 | D04, D10 |
| Operational reliability | D09 | D10, D11 |
| Risks and roadmap | D11 | D10, D12 |
| Evidence appendix | D12 | D10 |

## Citation Protocol

1. Every normative claim in the white paper must map to an evidence ID from D12.
2. If no evidence ID exists, classify as assumption and track in risk section.
3. Prefer primary-document citations over secondary references.

## Draftability Acceptance Check

A new writer passes draftability when they can produce:

1. A sectioned outline with at least seven top-level sections.
2. At least one evidence-linked claim per section.
3. Explicit risk and assumption sections referencing D11 and D12.
