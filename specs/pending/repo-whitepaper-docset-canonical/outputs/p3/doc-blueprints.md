# P3 Document Blueprints

## Objective

Freeze scope boundaries and outlines for D01-D12.

## Blueprint Summary

| Doc ID | Primary Question | Must Include | Must Exclude |
|---|---|---|---|
| D01 | How should readers navigate this corpus? | source map, reading paths, usage modes | deep technical implementation details |
| D02 | What is the executive technical thesis? | top-level narrative and differentiators | exhaustive schema listings |
| D03 | What terms and concepts are canonical? | glossary and term decisions | operations runbook details |
| D04 | How does the JSDoc semantic system work? | tag model, applicability, derivability | broad ops strategy |
| D05 | How does the codegraph architecture flow end-to-end? | components, dataflow, phase contracts | low-level tag unions |
| D06 | How do deterministic extraction and enrichment methods operate? | methods, constraints, validation loops | roadmap governance |
| D07 | How is NLP/reasoning strategy scoped and bounded? | modules, reasoning guardrails, status labels | infrastructure deployment specifics |
| D08 | What interfaces and contracts exist? | interface catalog, schema contracts, invariants | broad narrative synthesis |
| D09 | How does the system operate reliably? | controls, SLOs, failure modes, recovery | taxonomy debates |
| D10 | How is quality measured and evidenced? | metrics, formulas, audits, cadence | roadmap ownership |
| D11 | What risks and decisions remain? | risk register, owners, triggers, roadmap | deep method internals |
| D12 | How do claims trace to evidence? | matrix, claim ledger, integrity checks | high-level storytelling |

## Global Authoring Constraints

1. Every normative claim references evidence IDs.
2. Primary topic ownership is exclusive.
3. Cross-references are required when details are delegated.
4. Claims are labeled with implementation status when uncertain.
