# D04 — JSDoc Semantic System and Tag Taxonomy

## Tag Definition Model

The canonical JSDoc semantic system in this corpus is schema-first and discriminated by tag identity. Tag semantics are represented through structured definitions, typed tag-name enumeration, and tagged union decoding behavior.

## Category Taxonomy

The canonical taxonomy classes represented in the tag-value union are:

| Tag class | Primary role |
|---|---|
| StructuralTagValues | Structure and declaration semantics. |
| DocumentationTagValues | Human-facing explanatory semantics. |
| TSDocTagValues | TSDoc-specific semantics and documentation policy effects. |
| TypeDocTagValues | TypeDoc generation and rendering semantics. |
| InlineTagValues | Inline linking/reference semantics. |
| OrganizationalTagValues | Ownership and organizational placement semantics. |
| EventDependencyTagValues | Event/dependency coupling semantics. |
| AccessModifierTagValues | Visibility and boundary-control semantics. |
| TypeScriptTagValues | TypeScript-specific behavior semantics. |
| ClosureTagValues | Closure-compiler compatibility semantics. |
| RemainingTagValues | Residual canonical tags outside prior focused classes. |

## Applicability and AST Derivability

### Applicability rules

`ApplicableTo` includes explicit AST attachment surfaces such as function, method, class, interface, module, file, event, and `any` as a catch-all surface. This defines where each tag class can legally attach.

### Derivability rules

1. Deterministic derivability applies when a tag can be produced directly from AST and canonical schema definitions.
2. Bounded-uncertain derivability applies when semantic interpretation requires contextual or inferred understanding, but still must be status-labeled.
3. Unknown derivability conditions are not accepted as normative claims unless they are labeled as assumptions.

## Examples and Constraints

### Deterministic example

`TagValue` composes category sub-unions and finalizes with a `_tag` discriminant (`toTaggedUnion`), enabling deterministic decode routing by tag identity.

### Deterministic example with count constraints

The canonical tag namespace includes 113 tag names, and exhaustiveness artifacts define expected source-list counts for jsdoc3, tsdoc, typescript, closure, and typedoc groupings.

### Bounded-uncertain example

When source material distinguishes `implemented`, `specified`, and `conceptual` states, semantic conclusions beyond deterministic schema facts must carry the matching status label.

### Constraint notes

1. Source-tag snapshots are versioned (`schemaVersion: 1`) and must remain compatible with discriminated decode assumptions.
2. Metadata surfaces (`name`, `url`, `retrievedAt`) must preserve URL/date-time codec integrity.
