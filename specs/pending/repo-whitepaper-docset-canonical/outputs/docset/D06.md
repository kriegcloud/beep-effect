# D06 — Deterministic Extraction and Enrichment Methods

## Deterministic Extractor Model

This method document defines how source artifacts are transformed into stable, evidence-ready technical statements without collapsing certainty tiers.

## Fibration Method

The extraction approach is fibration-oriented:

1. Canonical tag definitions provide stable semantic anchors.
2. Tag-value union composition provides deterministic structural routing.
3. `_tag` discriminants preserve decode determinism.
4. Applicability surfaces constrain legal attachment contexts.
5. Metadata codecs preserve source provenance typing.

## Structured Narrowing

Structured narrowing proceeds in ordered passes:

1. Identify candidate tag-bearing source artifacts.
2. Decode through the discriminated `TagValue` union.
3. Narrow claims by applicability surfaces and status class.
4. Retain certainty-tier annotation.
5. Emit evidence-linked claim surfaces for downstream documents.

## Validation Loop

Validation is iterative and evidence-bound:

1. Compare extracted coverage against canonical expected counts.
2. Verify snapshot schema compatibility.
3. Confirm evidence ID presence for normative claim surfaces.
4. Escalate unresolved mismatches as explicit conflicts rather than silent drops.

## Boundary Conditions and Reproducibility Constraints

1. Reproducibility requires stable canonical tag definitions and tag-name literals.
2. Count-based validation must use declared expected source-group counts.
3. Snapshot interpretation must respect declared schema version.
4. Known path-mismatch behavior in validation script imports is treated as a deferred reliability boundary condition and must remain explicitly tracked.
5. Inferred enrichment beyond deterministic extraction must preserve status labels (`implemented`, `specified`, `conceptual`).

## Method Failure Modes

| Failure mode | Detection pattern | Response |
|---|---|---|
| Decode ambiguity | `_tag` does not map to expected variant behavior | Halt deterministic path; record bounded-uncertain interpretation in downstream reasoning docs. |
| Coverage drift | Expected counts do not match extracted sets | Flag quality issue and re-run narrow pass with explicit exclusions/assumptions. |
| Provenance loss | Metadata URL/date-time codec materialization missing | Reject claim as normative until provenance fields are restorable. |
| Script reliability mismatch | Import paths differ from artifact location | Keep as deferred conflict with owner/disposition tracking. |
