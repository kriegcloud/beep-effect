# D09 — Operations, Reliability, and Control Planes

## SLO/SLA Framing

| Objective | Definition | Target | Operational use |
|---|---|---|---|
| Request success rate | Successful extraction requests divided by total extraction requests. | Greater than or equal to 98%. | Used as the primary reliability SLO for release readiness. |
| Soft timeout rate | Soft stage timeouts divided by total stage executions. | Less than 2%. | Triggers stage-budget tuning and workload shaping. |
| Hard timeout rate | Hard stage timeouts divided by total stage executions. | Less than 0.1%. | Triggers incident triage and stage-level escalation review. |
| Queue depth pressure | Number of pending requests in control queue. | Alert above 20 queued requests. | Triggers throttle/retry/backpressure controls. |
| Concurrency bound | Concurrent requests at rate-limiter boundary. | Maximum 5 in balanced profile. | Protects upstream model endpoint reliability. |

## Failure Modes and Recovery

| Failure condition | Detection signal | Immediate action | Escalation |
|---|---|---|---|
| Stage timeout growth | Soft or hard timeout trend rises above thresholds. | Apply scope reduction and stage retry policy; preserve partial results. | Escalate to Ops/Reliability Lead if hard timeouts exceed target for two consecutive windows. |
| Queue pressure saturation | Queue depth warning at 80% then critical growth above max operating threshold. | Increase consumer rate, enable event sampling, and reduce inbound concurrency. | Escalate to platform on-call when queue remains saturated after one control window. |
| Token budget exhaustion | Request budget rejection or repeated budget-exceeded signals. | Rebalance stage allocations and reduce low-priority inference scope. | Escalate to Quality Lead when budget exhaustion causes measurable gate failure risk. |
| Rate-limiter circuit open | Consecutive upstream failures trip circuit breaker state. | Enter degraded mode, pause non-critical extraction paths, and preserve deterministic outputs. | Escalate as operational incident when circuit remains open beyond reset interval. |
| Script reliability mismatch | Validation script import path mismatch persists for canonical audit surface. | Treat as known reliability risk; do not mark as resolved operationally. | Escalate through D11 risk governance until remediation owner closes carry item. |

## Control Planes

### LLM control plane

1. Enforce per-stage timeout controls with bounded soft and hard limits.
2. Enforce per-request token budgeting with stage allocations and buffer.
3. Enforce centralized rate limiting with bounded concurrency and circuit behavior.
4. Preserve partial-result policy under timeout conditions instead of fabricating completion.

### Progress streaming control plane

1. Emit status and error events for lifecycle observability.
2. Monitor queue depth and inject backpressure warnings when pressure rises.
3. Apply sampling and queue strategies to keep stream durability under load.
4. Preserve cancellation and recovery semantics with explicit partial-result signaling.

### Persistence and idempotency control plane

1. Persist run metadata and audit events in deterministic run surfaces.
2. Preserve fire-and-forget audit emission to avoid blocking extraction completion paths.
3. Apply idempotency key discipline for deterministic replay and cache behavior.
4. Track unresolved reliability carries in governance surfaces rather than resolving by assumption.
