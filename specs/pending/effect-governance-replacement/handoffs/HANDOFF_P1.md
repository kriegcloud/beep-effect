# Effect Governance Replacement - P1 Handoff

## Goal

Validate research claims, reject weak options, and lock a fixed steering evaluation corpus plus a credible shortlist.

## Required Inputs

- [../RESEARCH.md](../RESEARCH.md)
- [../outputs/parity-matrix.md](../outputs/parity-matrix.md)
- [../outputs/steering-eval-corpus.md](../outputs/steering-eval-corpus.md)
- [../outputs/candidate-scorecard.md](../outputs/candidate-scorecard.md)
- live repo files needed to validate parity or capability claims

## Deliverables

- [../VALIDATED_OPTIONS.md](../VALIDATED_OPTIONS.md)
- expanded [../outputs/parity-matrix.md](../outputs/parity-matrix.md)
- locked [../outputs/steering-eval-corpus.md](../outputs/steering-eval-corpus.md)
- updated [../outputs/candidate-scorecard.md](../outputs/candidate-scorecard.md)

## Must Answer

- which options survive validation
- which options are rejected and why
- where parity is exact, partial, or not credible
- what the locked evaluation corpus is
- what remains unknown but still matters for planning

## Stop Conditions

- stop if the evaluation corpus cannot be locked credibly
- stop if parity claims cannot be grounded one rule at a time
- stop if the phase would need to mutate repo behavior outside this package
