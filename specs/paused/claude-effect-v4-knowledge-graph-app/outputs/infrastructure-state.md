## Overview

All infrastructure for the Knowledge Graph Explorer is provisioned via SST IaC. The SST configuration lives at the monorepo root and manages Railway, Neon, and Vercel resources through a single deploy command.

**Full infrastructure documentation:** `specs/pending/sst-infrastructure/`

---

## Deploy Command

```bash
op run --env-file=.env -- bunx sst deploy --stage dev
```

This single command:
1. Reads secrets from 1Password via `op://` references in `.env`
2. Deploys Railway project with 3 services (FalkorDB, Graphiti MCP, Caddy Auth Proxy)
3. Provisions Neon PostgreSQL database (`beep_auth` on PG 17)
4. Creates Vercel project with all 10 environment variables
5. Returns deploy outputs (project IDs, URLs, connection strings)

## Known Manual Steps (Railway Provider v0.4.4 Gaps)

These steps must be performed via the Railway dashboard after each fresh `sst deploy` to a new stage:

| Step | Reason | Status (dev) |
|------|--------|-------------|
| FalkorDB volume at `/data` | Volume creation fails in provider v0.4.4 | Done |
| Auth Proxy `*.up.railway.app` domain | ServiceDomain creation returns 400 | Done |
| Auth Proxy start command | No `startCommand` field on provider | Done |

**Start command for auth proxy:**
```
sh -c 'printf "%s" "$CADDYFILE" > /etc/caddy/Caddyfile && caddy run --config /etc/caddy/Caddyfile --adapter caddyfile'
```

**Full details:** `specs/pending/sst-infrastructure/outputs/p1-railway-provider-gaps.md`

## How to Verify Infrastructure is Running

```bash
# 1. Check Railway auth proxy (no API key â†’ 401)
curl -s -o /dev/null -w "%{http_code}" https://auth-proxy-production-91fe.up.railway.app/mcp
# Expected: 401

# 2. Check Railway auth proxy (with API key â†’ 200)
curl -s -H "X-API-Key: <GRAPHITI_API_KEY>" https://auth-proxy-production-91fe.up.railway.app/mcp
# Expected: 200 (Graphiti Agent Memory SSE response)

# 3. Check Vercel deployment
curl -s -o /dev/null -w "%{http_code}" https://beep-dev.vercel.app
# Expected: 200 (or 307 redirect to sign-in)

# 4. Check Neon database tables
op run --env-file=.env -- npx tsx -e "
import { neon } from '@neondatabase/serverless';
const sql = neon(process.env.DATABASE_URL_UNPOOLED!);
(async () => {
  const t = await sql\`SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' ORDER BY table_name\`;
  console.log('Tables:', t.map(r => r.table_name));
})();
"
# Expected: [ 'account', 'session', 'user', 'verification' ]
```
