# D10 — Validation, Metrics, and Audit Evidence

## Metric Catalog

D10 defines measurable quality for the corpus and links each quality claim to an evidence route. All quality interpretation in this document remains bounded by certainty tiers and status labels.

| Metric ID | Metric | Purpose |
|---|---|---|
| M-001 | NL-to-Cypher accuracy | Validate semantic query translation reliability. |
| M-002 | Sub-graph p95 latency | Validate interactive query performance bound. |
| M-003 | Vector recall@10 | Validate retrieval relevance quality. |
| M-004 | Normative evidence linkage rate | Validate evidence integrity for claims. |
| M-005 | Operational success rate | Validate runtime extraction reliability posture. |
| M-006 | Timeout quality envelope | Validate soft/hard timeout behavior against target. |

## Formula Definitions

| Metric ID | Formula | Target |
|---|---|---|
| M-001 | `NL-to-Cypher Accuracy = (correct_queries / total_queries) * 100` | Greater than or equal to 80% |
| M-002 | `Sub-graph p95 latency = percentile(latency_ms, 95)` for two-hop extraction queries | Less than or equal to 200 ms |
| M-003 | `Recall@10 = (relevant_results_in_top10 / total_relevant_results) * 100` | Greater than or equal to 75% |
| M-004 | `Normative evidence linkage rate = (linked_normative_claims / total_normative_claims) * 100` | 100% |
| M-005 | `Request success rate = (successful_requests / total_requests) * 100` | Greater than or equal to 98% |
| M-006 | `Soft timeout rate = (soft_timeouts / stage_executions) * 100` and `Hard timeout rate = (hard_timeouts / stage_executions) * 100` | Soft less than 2%; hard less than 0.1% |

## Audit Mapping

| Audit or gate artifact | Linked metrics | Evidence route |

| Certainty lock defaults | M-004 interpretation boundaries | E-S02-001 -> status/certainty constrained scoring notes |
| Status-label architecture discipline | M-004 interpretation boundaries | E-S03-001 -> status-aware quality reporting policy |

## Evaluation Cadence

1. Per-run checks: M-005 and M-006 at extraction runtime boundaries.
2. Daily checks: M-002 latency and M-006 timeout envelope trend.
3. Weekly checks: M-001 and M-003 quality trend with sampled query suite.
4. Phase-gate checks: M-004 must be exactly 100% before P6 promotion.
5. Incident-triggered checks: re-evaluate M-005 and M-006 immediately after reliability incidents.

## Delegation Rules

1. Runtime control implementation details and operational action paths are delegated to D09.
2. Risk ownership, due dates, and decision escalation are delegated to D11.
3. Full evidence ledger and claim-level linkage accounting are delegated to D12.
4. Executive synthesis of quality significance is delegated to D02.
