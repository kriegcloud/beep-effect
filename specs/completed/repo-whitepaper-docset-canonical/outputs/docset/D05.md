# D05 — Codegraph Architecture and Dataflow

## System Components

The architecture is composed as a phase-governed technical corpus pipeline with explicit boundaries between source ingestion, semantic modeling, deterministic extraction, reasoning overlays, contract publication, and operational governance.

## End-to-End Dataflow

### Stage A: Source-area ingestion

All four locked source areas are treated as the corpus input boundary. Source artifacts are not normalized away from their original path context.

### Stage B: Fact and evidence seeding

Evidence-linked facts establish stable references that downstream documents consume through evidence ID pointers.

### Stage C: Taxonomy and terminology normalization

Canonical term and taxonomy decisions constrain language and categorization before narrative drafting.

### Stage D: Core technical drafting

D01-D08 provide primary ownership coverage across navigation, synthesis, semantics, architecture, methods, reasoning, and contracts.

### Stage E: Governance and traceability completion

D09-D12 finalize operations, metrics, risk governance, and source-to-claim integrity.

### Stage F: Quality gating and publication handoff

P6 and P7 certify consistency, completeness, and publication readiness.

## Certainty Layers

The architecture uses a three-tier certainty layer contract:

1. Deterministic certainty (`1.0`): derived directly from deterministic source artifact structures.
2. Type-system certainty (`0.85`-`0.95`): derived from typed schema and compile-time semantics.
3. LLM-inferred certainty (`0.6`-`0.85`): derived from bounded inference and must remain explicitly labeled.

## Phase Contracts

| Phase | Contracted output behavior | Promotion requirement |
|---|---|---|
| P0 | Governance and package structure are established. | Required artifacts exist and parse. |
| P1 | Source inventory and seeded evidence IDs are complete. | Coverage baseline and fact ledger integrity checks pass. |
| P2 | Terminology/taxonomy conflicts are resolved or explicitly deferred. | No blocker-level unresolved conceptual conflict. |
| P3 | Document boundaries and ownership are frozen in manifest and blueprints. | One-to-one primary-topic ownership is maintained. |
| P4 | D01-D08 drafts are authored with evidence-linked normative claims. | Completion checks and internal consistency pass. |
| P5 | D09-D12 finalize operations, metrics, governance, and traceability. | High-impact claims are evidence-linked. |
| P6 | Pairwise consistency and completeness gates are certified. | All rubric dimensions pass. |
| P7 | Starter kit and publication handoff package are finalized. | Corpus is draftable without external discovery. |

## Dependency and Control-Flow Notes

2. P5 depends on stabilized claim surfaces from P4.
3. P6 depends on full D01-D12 availability.
4. P7 depends on P6 gate pass and traceability completeness.
