This document onboards readers to the corpus and defines how to navigate D01-D12 without scope drift.

## Reading Paths by Persona

### Persona A: New contributor (fast onboarding)

1. Start with D01 for corpus map and ownership boundaries.
2. Read D03 for canonical term usage before interpreting technical claims.
3. Read D05 for architecture flow and phase contracts.
4. Read D04 and D08 for semantic model plus interface contracts.

### Persona B: Architecture lead (system decisions)

1. Start with D05 for component boundaries and certainty layers.
2. Read D06 for deterministic extraction and validation mechanics.
3. Read D08 for interface invariants and ownership expectations.
4. Read D09 and D11 for operational and governance constraints.

### Persona C: White-paper author (narrative synthesis)

1. Start with D02 for thesis and differentiator framing.
2. Read D01 to preserve canonical reading pathways and delegation rules.
3. Read D10 for quality gates and measurable claims.
4. Read D12 for evidence ID mapping during drafting.

### Persona D: Reviewer/compliance editor (evidence and consistency)

1. Start with D03 for canonical language policy.
2. Read D12 for traceability-link expectations.
3. Read D10 for quality-gate semantics.
4. Cross-check D02, D05, D08 claims against their evidence tables.

## Usage Modes

### Mode 1: Orientation pass

Use D01 + D03 only to establish corpus structure and canonical term usage.

### Mode 2: Technical implementation pass

Use D05 + D06 + D08 as the backbone, then D04 and D07 for semantic and reasoning constraints.

### Mode 3: Publication pass

Use D02 + D10 + D12 for thesis, quantitative quality, and evidence-linked validation.
