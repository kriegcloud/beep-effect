# Phase 0: Parity Audit (`beep purge`)

## Goal
Implement `beep purge` in `@beep/repo-cli` (`tooling/cli`) with Effect v4 and command-tree conventions, while matching legacy purge intent.

## Legacy Behavior Map
Source: `.repos/beep-effect/tooling/repo-scripts/src/purge.ts`

### Legacy purge target sets
- Workspace artifact names:
  - `.tsbuildinfo`
  - `build`
  - `dist`
  - `.next`
  - `coverage`
  - `.turbo`
  - `node_modules`
- Root artifacts removed on every run:
  - `node_modules`
  - `.turbo`
- Root artifact removed only with lock flag:
  - `bun.lock`

### Legacy target discovery
- Repo root via `findRepoRoot`.
- Workspace directories via **hardcoded glob patterns** (`WORKSPACE_PATTERNS`) + `glob.sync`.
- Per-workspace artifact paths are generated by joining each workspace dir with the workspace artifact names.

### Legacy execution semantics
- Each target removed with `fs.remove(target, { recursive: true, force: true })`.
- Missing paths are tolerated (idempotent cleanup).
- Logs:
  - per-target “Preparing to remove …”
  - summary “Removing X paths …”
- Runs removals concurrently.

## Current CLI Architecture Audit (`tooling/cli`)

### Command model
- Uses `effect/unstable/cli` (`Command`, `Flag`, `Argument`).
- Commands are module-level exports from `tooling/cli/src/commands/*`.
- Root tree assembled in `tooling/cli/src/commands/root.ts` using `Command.withSubcommands`.
- Package exports curated in `tooling/cli/src/index.ts`.

### Runtime/layers
- Entry point: `tooling/cli/src/bin.ts`.
- Provides Node services (`NodeFileSystem`, `NodePath`, `NodeTerminal`) plus `FsUtilsLive`.
- Commands can depend on repo-utils helpers/services directly (`findRepoRoot`, `resolveWorkspaceDirs`, etc.).

### Existing command patterns
- Use `Effect.fn(function* () { ... })` handlers.
- Use `Console.log`/`Console.error` for command output.
- Prefer tagged errors from `@beep/repo-utils` (`DomainError`, etc.) instead of untyped `Error`.

### Test harness pattern
- Tests run command effects through `Command.runWith(...)` where needed.
- Test layers merge Node services + `TestConsole` + mocked `ChildProcessSpawner` + `FsUtilsLive`.
- Temp filesystem fixtures are used for mutation-heavy behavior.

## Parity Gaps Identified
- `purge` command does not exist in `tooling/cli`.
- `rootCommand` does not register `purge`.
- `index.ts` does not export `purgeCommand`.
- No purge-focused tests exist.
- Legacy workspace discovery is static and unsuitable for current spec requirement.

## Workspace Discovery Strategy Decision
Decision: derive workspace directories from current repo config using repo-utils.

- Root lookup: `findRepoRoot()`.
- Workspace lookup: `resolveWorkspaceDirs(rootDir)` from `tooling/repo-utils/src/Workspaces.ts`.
  - This reads `root/package.json` `workspaces` globs.
  - Expands globs through `FsUtils`.
  - Returns concrete package dirs from actual workspace `package.json` files.

Rationale:
- Satisfies hard requirement to avoid legacy hardcoded workspace globs.
- Tracks repo changes automatically as workspaces evolve.
- Reuses existing Effect v4 repo utilities and error model.
