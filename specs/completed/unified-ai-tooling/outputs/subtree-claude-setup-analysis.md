# Summary
- The `.claude/settings.json` hooks block wires `SessionStart` → `agent-init`, `UserPromptSubmit` → `skill-suggester`, and `Pre/PostToolUse` → `subagent-init`/`pattern-detector`, giving a template for layered bootstrapping and guards (`settings.json:14`).
- `agent-init` runs `tree`, git, module/architecture crawlers, GH queries, and `mise` tasks before emitting a richly structured `<session-context>`, while `subagent-init` reuses the module summary + `mise` tasks to keep fast implementer prompts (`hooks/agent-init/index.ts:116`, `hooks/agent-init/index.ts:269`, `hooks/subagent-init/index.ts:154`).
- The `skill-suggester` hook manages `.claude/.hook-state`, fetches mise tasks, scores skills, and folds in `context-crawler` output so each prompt can show `<system-hints>` with relevant modules and gating reminders; the same crawler is exposed through the `/modules` command for easy discovery (`hooks/skill-suggester/index.ts:40`, `hooks/skill-suggester/index.ts:87`, `hooks/skill-suggester/index.ts:506`, `scripts/context-crawler.ts:1`, `commands/modules.md:1`).

# Key Patterns to Reuse
- `context-crawler.ts` walks every `ai-context.md`, summarizes modules, and powers both commands and session hooks so agents always have a distilled module view without manual searching (`scripts/context-crawler.ts:1`, `commands/modules.md:1`, `hooks/agent-init/index.ts:198`).
- The pattern stack keeps `.claude/patterns/{code-smells,dangerous-commands,custom}` documented, provides a template for new rules, and feeds `pattern-detector` which sorts permissions by severity and tolerates running from within `.claude` via `CLAUDE_PROJECT_DIR` detection (`patterns/README:5`, `patterns/TEMPLATE:1`, `hooks/pattern-detector/index.ts:1`, `hooks/pattern-detector/core.ts:73`).
- onboarding/docs include a Quick Start that walks through testing, creating, debugging, and migrating patterns plus a skill chapter that explicitly champions `Config.redacted` for API keys, giving a repeatable training loop (`hooks/QUICKSTART.md:1`, `hooks/QUICKSTART.md:182`, `skills/effect-ai-provider/SKILL.md:67`).

# Risks/Anti-patterns
- Commands that `cat`/`less` `.env`, `.pem`, or `.key` files are elevated to a “high-ask” pattern so leaking secrets through terminal output is caught early (`patterns/dangerous-commands/env-secrets.md:1`).
- The documentation flags `Command.env` with literal `API_KEY` values as a hard-coded secret anti-pattern, so agents should never mimic that snippet without injecting `Config.redacted` (`skills/platform-abstraction/SKILL.md:385`).
- Neglecting to migrate `.claude/smells`/`.claude/dangerzone` into the new structure leaves stale hooks and loses coverage, which is why the Quick Start explicitly runs `migrate-patterns` and archives the old dirs (`hooks/QUICKSTART.md:182`).
- Invalid frontmatter (missing `name`, `description`, `pattern`) or malformed YAML silently prevents patterns from loading, so the Quick Start stresses strict formatting and validation before relying on the detector (`hooks/QUICKSTART.md:208`).
- Running the detector from inside `.claude` without canonicalizing the project root breaks pattern discovery, so the loader first checks `process.cwd()` and falls back to `CLAUDE_PROJECT_DIR` before reading `.claude/patterns` (`hooks/pattern-detector/core.ts:73`).

# Concrete Recommendations for beep-sync
- Copy the lifecycle wiring from `settings.json` and build `agent-init`/`subagent-init` scripts that gather tree/git/module/architecture state plus `skill-suggester` reminders so every session has the same rich bootstrap and the gelled guardrails (`settings.json:14`, `hooks/agent-init/index.ts:116`, `hooks/agent-init/index.ts:269`, `hooks/subagent-init/index.ts:154`, `hooks/skill-suggester/index.ts:40`).
- Introduce a crawler like `context-crawler.ts` plus `/modules`-style commands so agents can list, inspect, and search `ai-context.md` docs without relearning the structure (`scripts/context-crawler.ts:1`, `commands/modules.md:1`).
- Adopt the documented `.claude/patterns` layout, the template, and the detector’s severity-sorted Pre/Post logic with `CLAUDE_PROJECT_DIR` detection to keep pattern maintenance and compatibility centralized (`patterns/README:5`, `patterns/TEMPLATE:1`, `hooks/pattern-detector/index.ts:1`, `hooks/pattern-detector/core.ts:73`).
- Ship a Quick Start that spells out how to test, create, debug, and migrate patterns (including the CLI script and archive steps) so new maintainers know exactly how to extend or clean the guardrails (`hooks/QUICKSTART.md:1`).
- Guard secrets by codifying the `Config.redacted` guidance into `skills`/docs and by adding detection rules similar to `env-secrets` to block direct cat/less/echo commands on secret files (`skills/effect-ai-provider/SKILL.md:67`, `patterns/dangerous-commands/env-secrets.md:1`).
